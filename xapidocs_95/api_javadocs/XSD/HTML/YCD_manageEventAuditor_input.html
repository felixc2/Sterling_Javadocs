Activeflag (maximum length: 1) yfs_event_auditor.active
Y/N flag indicating if this auditor event is active and should perform the action it corresponds to.
ApplicationCodestring (maximum length: 50) yfs_event_auditor.application_code
This is the corresponding application code for the wizard id provided and is used to add notes on confirmation of a wizard flow.
EventAuditorKeykey (maximum length: 24) yfs_event_auditor.event_auditor_key
The primary key of the YFS_EVENT_AUDITOR table.
EventAuditorTypestring (maximum length: 24) yfs_event_auditor.event_auditor_type
Determines the type of auditor event this record corresponds to. Valid values are: TRANSACTION_EVENT, HOLD_ADDED_EVENT, HOLD_RESOLVED_EVENT, HOLD_REJECTED_EVENT, or APPLICATION_EVENT.
HoldTypestring (maximum length: 20) yfs_event_auditor.hold_type
If this auditor record corresponds to a hold event, this column stores the hold type that this record corresponds to.
NoteTextstring (maximum length: 2000) yfs_event_auditor.note_text
If provided, this field will be used to set the text in a note.
NoteTypestring (maximum length: 40) yfs_event_auditor.note_type
The note type that will be logged in the notes table (stored in the notes table as ReasonCode).
OrganizationCodestring (maximum length: 24) yfs_event_auditor.organization_code
The organization code assocaited with this event auditor.
This attribute corresponds to the OrganizationCode Fact.
TransformationFilestring (maximum length: 100) yfs_event_auditor.transformation_file
The XSL file that will be used to transform the XML data to the notes text.
WizardIDstring (maximum length: 255) yfs_event_auditor.wizard_id
