ElementsLoadStop
LoadStop Associated With Table: YFS_LOAD_STOP
Stores information about a load stop. A load stop is identified by a stop node - which is a receiving or shipping node on the system, or a stop address in cases where stop nodes do correspond to a receiving node which is not maintained on the system. A load stop is created and managed by the manageLoad API. Certain time related attributes such as actual arrival and departure dates, and the stop reached attribute are updated by the receiveIntransitUpdates API.

StopSequenceNononNegativeInteger (total number of digits: 5) yfs_load_stop.stop_sequence_no
The stop sequence number that determines the load travel path. The origin stop is assigned the sequence no. -1 and the destination stop is assigned the sequence no. 99999. All intermediate stops may be sequenced by the user to fall within this range. Stop Type attribute of the stop determines whether a stop is the origin, destination or an intermediate stop.
StopTypestring (maximum length: 1) yfs_load_stop.stop_type
The type of the load stop. There are three types of load stops: O - corresponds to the origin stop, D - corresponds to the destination stop and I - for an intermediate stop.
