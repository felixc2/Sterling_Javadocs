ChangeProject Associated With Table: YFS_CHANGE_PROJECT
This table stores Change Project definitions. A Change Project provides the mechanism to consolidate database changes under a single unit. A Change Project can be approved on one system so that the database changes associated with it can be sent to another system. A Change Project is a collection of Change Requests. It is also the unit which a valid user can approve or reject. All data changed as part of a Change Project will be logged under the same Change Project.
AttributeTypeRequiredTable.Column
ChangeProjectIDstring (maximum length: 24) yfs_change_project.change_project_id
The identifier for the Change Project. If it is not passed during creation, it is generated and pre-fixed with the Deployment Type prefix. It cannot be set to DEFAULT.
ChangeProjectKeykey (maximum length: 24) yfs_change_project.change_project_key
Primary key attribute of YFS_CHANGE_PROJECT.
IsImportedflag (maximum length: 1) yfs_change_project.is_imported
If set as Y, indicates whether this Change Project was imported from another instance.
LongDescriptionstring (maximum length: 255) yfs_change_project.long_description
The long description of the Change Project.
ManageUserIDstring (maximum length: 50) yfs_change_project.manage_user_id
Identifier for the user who caused a change in the Change Project. This is always taken from the Environment.
OrganizationCodestring (maximum length: 24) yfs_change_project.organization_code
The organization that defines the Change Project.
ShortDescriptionstring (maximum length: 100) yfs_change_project.short_description
The short description of the Change Project.
Statusstring (maximum length: 10) yfs_change_project.status
The status of the Change Project. As the Change Project progresses through the change process, the status changes.
