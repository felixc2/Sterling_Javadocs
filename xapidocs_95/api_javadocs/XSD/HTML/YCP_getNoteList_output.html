Note Associated With Table: YFS_NOTES
Generic table used to stores notes associated with various entities in the system.
AttributeTypeRequiredTable.Column
AuditTransactionIdstring (maximum length: 40) yfs_notes.audit_transaction_id
The identifier of the order modification audit that caused this record.
ContactReferencestring (maximum length: 150) yfs_notes.contact_reference
If the notes were entered because of customer contact by phone, the phone no can be stored in this column. If the contact is by email, email ID can be stored here.
ContactTimedateTimeyfs_notes.contact_time
The time at which the customer contact occurred.
ContactTypestring (maximum length: 100) yfs_notes.contact_type
The contact type like Phone call, email etc. The values will be selected from common code of code type "CONTACT_TYPE"
ContactUserstring (maximum length: 40) yfs_notes.contact_user
The user who made the contact. The default value will the logged in user.
CustomerSatIndicatorstring (maximum length: 255) yfs_notes.customer_sat_indicator
The customer satisfaction level.
NoteTextstring (maximum length: 2000) yfs_notes.note_text
Detailed Notes.
Prioritydecimal (minimum value: 0
total number of digits: 10
fraction digits: 2) 		yfs_notes.priority
Priority of the note. 1 is High, 0 is Low.
ReasonCodestring (maximum length: 40) yfs_notes.reason_code
The reason for which the notes are logged. Like "CustomerContact" etc. The values will be selected from common code of code type "NOTES_REASON"
SequenceNononNegativeInteger (total number of digits: 5) yfs_notes.sequence_no
The sequence of the note.
Tranidstring (maximum length: 40) yfs_notes.tranid
Transaction for which these notes are applicable.
Trannamestring (maximum length: 50) .
Specifies the name of the Transaction
VisibleToAllflag (maximum length: 1) yfs_notes.visible_to_all
This flag determines whether the note applied should be visible to all users. This field is used for filtering. No logic is currently applied to it.
