ElementsOrder
Order Associated With Table: YFS_ORDER_HEADER
To identify the order that shipments belong to, pass either the OrderHeaderKey or the EnterpriseCode, DocumentType and OrderNo.
To identify the release that shipments belong to, pass either the OrderReleaseKey, a combination of ReleaseNo and OrderHeaderKey, or a combination of EnterpriseCode and ReleaseNo.
To identify the order line that shipments belong to, pass either the OrderLineKey, PrimeLineNo, or SubLineNo.

This element corresponds to the IndexLookup Fact. The contents of this element can be used for for searching against index.

OrderHeaderKey or EnterpriseCode, DocumentType and OrderNo must be passed and must be valid to identify an order.

OrderLineKey, PrimeLineNo, or SubLineNo must be passed and must be valid to identify an order release.

ConsiderOnlyDropShippedOrdersstring (maximum length: 1) .
If set to Y then chained orders will only be considered if they are drop shipments. Any shipments on the original order will still be considered regardless.
DocumentTypestring (maximum length: 40) yfs_order_header.document_type
The document type that is associated with this order. For example, Sales Order document (0001), Planned Order document (0002), Return Order document (0003) , and Purchase Order document (0005). This field determines the order's fulfillment cycle and controls the inventory and payment updates performed on the order.
This attribute can be used for searching against Order index.

ReadFromHistorystring (maximum length: 1) .

Valid Values:
N - Returns records that satisfy the input criteria from the YFS_SHIPMENT table (the current table).
Y - Returns records that satisfy the input criteria from the YFS_SHIPMENT_H table (the history table).
B - Returns records that satisfy the input criteria from both YFS_SHIPMENT and YFS_SHIPMENT_H tables. Records returned from the current table are first in the output.

If a value is not passed, it will be defaulted based upon the order uniquely identified from the input attributes. If the order is a history order, this value will default to "Y", else to "N".
