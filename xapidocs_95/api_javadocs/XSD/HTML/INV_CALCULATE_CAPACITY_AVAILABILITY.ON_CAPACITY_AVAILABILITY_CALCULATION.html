ResourcePool Associated With Table: YFS_RES_POOL
This table stores information for a resource pool. A resource pool represents a collection of resources for providing services. The resources represented by a resource pool should have common characteristics (like regions serviced and skill sets). Capacity for providing services is maintained at the Resource Pool level.
AttributeTypeRequiredTable.Column
AllRegionsServicedflag (maximum length: 1) yfs_res_pool.all_regions_serviced
This flag indicates whether all regions are serviced by this resource pool.
CalendarKeykey (maximum length: 24) yfs_res_pool.calendar_key
The calendar that is associated with this resource pool.
CapacityInfoAvailableflag (maximum length: 1) yfs_res_pool.capacity_info_available
This flag indicates whether this resource pool has capacity information available.
CapacityMaintainAtResourceLevelflag (maximum length: 1) yfs_res_pool.cap_maintain_at_resource_level
This flag indicates whether capacity is maintained at the resource level for this resource pool. Default Value is 'N'. If set to Y, resource pool is allowed to define resources, and Standard and/or Override capacity cannot be defined for the resource pool.
CapacityOrganizationCodekey (maximum length: 24) yfs_res_pool.capacity_organization_code
A Resource Pool is always created wihin the context of a Capacity Organization. An Organization can source services from any Resource Pools within the same Capacity Organization.

CapacityUnitOfMeasurestring (maximum length: 40) yfs_res_pool.capacity_uom

ItemGroupCodestring (maximum length: 20) yfs_res_pool.item_group_code
The type of item that this resource pool services.
Allowed ValueDescription

SS Indicates that this resource pool is used for special service items. 
PROD Indicates that this resource pool is used for product items.
