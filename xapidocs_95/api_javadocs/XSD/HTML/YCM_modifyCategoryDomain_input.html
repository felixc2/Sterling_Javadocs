CategoryDomain Associated With Table: YFS_CATEGORY_DOMAIN
Highest level of classification of an item. Multiple domains can be created by an organization. All categorization is defined under a particular domain.

Validation ConditionError
Either the CategoryDomainKey attribute or the combination of the OrganizationCode, CategoryDomain, and IsClassification attributes must be passed and must be valid.  
If not passed:
YCM0002Mandatory parameters for the operation are missing
If not valid:
YCM0034Category Domain Not Found
The CategoryDomain, OrganizationCode, and IsClassification attributes cannot be modified. If modification is attempted:
YCM0040Primary Key or Unique Keys cannot be modified
If AuthorizedSubCatalogOrganizationCode is not of a valid sub-catalog organization of the catalog organization specified in OrganziationCode field:
YCM83_007Invalid sub-catalog organization

AttributeNamestring (maximum length: 100) yfs_category_domain.attribute_name
A pre-defined list of fields that can be used for item classification.

Validation ConditionError
If IsClassification is set to "Y", AttributeName must be passed. If not passed:
YCM0002Mandatory parameters for the operation are missing
If AttributeName is passed, it must be a valid attribute name of an item that the system allows. The valid values are HarmonizedCode, Schedule_B_Code, NMFCCode, NMFCClass, ECCNNo, NAICSCode, UNSPSC, HazmatClass, ProductLine, VelocityCode, TaxProductCode, ItemType, StorageType, PickingType, CommodityCode, and extended attributes of Item. If not a valid attribute name:
YCM0050Invalid Item Classification Attribute
