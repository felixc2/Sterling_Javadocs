VarianceTypestring (maximum length: 20) yfs_count_result.variance_type
This field determines the variance type based on the count results. If the Variance Quantity > 0, the Variance Type for the items will be 'New', 'Missing' if the Variance Quantity < 0, and 'No Variance' if the Variance Quantity = 0. However, for License Plates the Variance Type will not be computed based on the Variance Quantity. Instead, the Variance Type will be set to 'New' if a Case/Pallet is found in the location, but does not exist in the system inventory of the location, 'Missing' if the Case/Pallet is not found in the location, but exists in the system inventory of the location, 'No Variance' in situations when there is no count discrepancy.

This field is computed as the summation of VARIANCE_QUANTITY * UNIT_COST for all tasks in the count request for the latest completed iteration.

This field indicates the type of the listener transaction. Each process type to process type combination consistutes a different listener type.
Allowed ValueDescription
01 Indicates the chained order to sales order listener type. 
02 Indicates the derived order to sales order listener type. 
03 Indicates the shipment to order listener type. 
04 Indicates the load to shipment listener type. 
05 Indicates the receipt to shipment listener type. 
06 Indicates the receipt to order listener type. 
07 Indicates the work order to container listener type. 
08 Indicates the work order to order listener type.
