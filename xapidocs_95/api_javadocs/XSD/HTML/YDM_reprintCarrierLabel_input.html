NoOfCopiesinteger (total number of digits: 9) .
Specify the number of copies you want to print, it should be a positive integer, default value of this attribute is 1
ReprintLabelLevelstring.
Valid values for this attribute are : FORWARD, RETURN, BOTH_FWD_AND_RET, PICKUP_SUMARY. If left blank, FORWARD is understood as the value of this attribute. If value FORWARD is given for a container, only Forward carrier label will be re-printed. If value RETURN is given for a container, only all the Return carrier labels will be re-printed. If value BOTH_FWD_AND_RET is given for a container, both the forward carrier label as well as all the return carrier labels will be re-printed. If value PICKUP_SUMARY is given, the manifest details should be passed in the input, and the manifest should be closed. Pickup Summary Label will be printed. This is applicable only for Pierbridge Integration. Although, if some specific Return Tracking numbers are selected to re-print their return labels, then irrespective of the value in the attribute, only those return labels will be printed.

Load Associated With Table: YFS_LOAD
Stores information about loads. Loads are made of Shipments, carry one or more shipments between two points, and carry an entire shipment. A load has one origin, one destination and can have multiple intermediate stops. A load has one carrier. Loads may or may not belong to a Delivery Plan. A load can pickup a shipment from its origin or an intermediate stop and drop off the shipment to its destination or an intermediate stop. A load is created and managed by the manageLoad API.
AttributeTypeRequiredTable.Column
LoadNostring (maximum length: 40) yfs_load.load_no
The load number.A unique key or business key, which by itself identifies a load.
Manifest Associated With Table: YFS_MANIFEST
Stores carrier manifest created during the manifesting process. A manifest is a carrier specific document that comprises of shipments. A manifest is created for a ship node and carrier. At any point in time, there is only one open manifest for a ship node and carrier. Every parcel shipment that is containerized is included in a manifest. Containerization is done using the packShipment and changeShipment API. All shipments included in the manifest are ship confirmed using the closeManifest API. All parcel related carrier services for a carrier are part of a single manifest.
