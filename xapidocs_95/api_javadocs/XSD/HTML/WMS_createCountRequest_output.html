CountRequest Associated With Table: YFS_COUNT_REQUEST
This transaction table stores count requests to perform inventory count in a warehouse. The count process in a warehouse begins with the count request.
AttributeTypeRequiredTable.Column
AisleNumberinteger (total number of digits: 9) yfs_count_request.aisle_number
This field indicates the aisle Number of the location to be counted. The locations belong to zones which have travel aisle's between them. A zone could belong to multiple aisles and multiple zones could belong to an aisle. But a location in a zone always belongs to one and only one aisle.
BayNumberinteger (total number of digits: 9) yfs_count_request.bay_number
This field indicates the location's bay number that is to be counted. Typically, the aisle, level and bay put together represents the physical location of the location in a node, if they are based on coordinate system. The bay attribute of the location (x-coordinate from the beginning of the aisle) is used in arriving at locations nearest to the dedicated locations algorithm that is used in putaway. Typically, the bay attribute is contained within the location ID.

LevelNumberinteger (total number of digits: 9) yfs_count_request.level_number
This field indicates the level number of the location to be counted. The height of the location (y-co-ordinate of the location from the floor) is classified as levels. The level attribute of the location is used in arriving at locations nearest to the dedicated locations algorithm used during the putaway process. Typically, the level attribute is contained within the location ID.
