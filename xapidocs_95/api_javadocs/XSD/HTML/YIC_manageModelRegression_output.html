Exclude

Used to filter the elements that are returned from the API. The API filters an element if:

  • The element doesn't match any of the inclusion specifiers

    • OR
  • The element matches one of the exclusion specifiers
The API doesn't filter Optionclass
 and Optionitem elements if it has child elements.
For eg. To filter Optionitem element(s) which has a set of properties associated to it, both Optionitem and Property has to be excluded by specifying exclude filters in the following manner:
<Exclude Class="Optionitem"/>
<Exclude Class="Property"/>

ManageModelRegression
The root xml element of the header.
AttributeTypeRequiredTable.Column

Description of the test case.

The name that identifies the group of test cases.

Operationstring (maximum length: 480) .
Indicates the operations that can be performed. The valid operations are:
  • Trap: Trap the test case by capturing a screen shot of the user selections.
  • Execute: Execute the test case identified by theTestKey.
  • Find: Search for test case(s) by TestId, GroupId or a combination of both TestId and GroupId. All the test cases will be listed, if both TestId and GroupId are not provided.
  • Rebase: Replace the existing baseline with the current response.
  • Debug: Execute the test case and return the rule firing trace.
  • GetXML[Report/Response/Request/Baseline]: Retrieves the Report, Response, Request or Baseline for the test case identified by theTestKey.

TestIdstring (maximum length: 480) .
The name that identifies the test case.
