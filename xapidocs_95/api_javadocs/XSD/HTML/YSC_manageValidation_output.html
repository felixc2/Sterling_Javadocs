Validation Associated With Table: YFS_VALIDATION
This table stores the configuration for a validation
AttributeTypeRequiredTable.Column
CanApprovalBePostponedflag (maximum length: 1) yfs_validation.approval_postponable
The flag indicates if the transaction violation can be approved at a later point of time. If it is marked as (N), then violation has to be approved before operator can proceed with the transaction.
ConfigurationGroupstring (maximum length: 40) yfs_validation.configuration_group
The configuration group that identify the level for which the validation can be configured.
Descriptionstring (maximum length: 100) yfs_validation.description
The description of the validation.
Domainstring (maximum length: 100) yfs_validation.domain
The validation domain for this configuration.
ErrorCodestring (maximum length: 24) yfs_validation.error_code
This specifies the error code that needs to thrown in case validation fails.
ExternalInitPerformflag (maximum length: 1) yfs_validation.external_init_perform
The flag indicates that the initial validation is done outside of the framework..
InputTemplatestring (maximum length: 40) yfs_validation.input_template
The template name identfier is for determining template to be used in business API for sending the data in XML format to approval engine.
IsApprovalRequiredOnReoccurenceflag (maximum length: 1) yfs_validation.reapproval_required
This flag specifies if each transaction violation for this validation needs to be approved or not.
IsOverridableflag (maximum length: 1) yfs_validation.is_overridable
This flag indicates that the validation can be overridden by overridden rules.
ModelTemplatestring (maximum length: 40) yfs_validation.model_template
The model template is used in SBC for showing appropriate model in Approval Rule creation screens
OrganizationCodestring (maximum length: 24) yfs_validation.organization_code
The organization for which the validation belongs.
ValidationIDstring (maximum length: 40) yfs_validation.validation_id
The unique identifier for the VALIDATION.
ValidationKeykey (maximum length: 24) yfs_validation.validation_key
The primary key of the YFS_VALIDATION table.
ValidationTypestring (maximum length: 10) yfs_validation.validation_type
The type of the validation.
ViolationEntitystring (maximum length: 100) yfs_validation.violation_entity
This specifies the violation entity for storing the violations and approver information.
ValidationReference Associated With Table: YFS_VALIDATION_REFERENCE
This table stores the configuration for a Validation Reference. Validation References are used to differentiate one violation from another.
AttributeTypeRequiredTable.Column
DataTypestring (maximum length: 40) yfs_validation_reference.data_type
Species the data type of the reference value.
IsPrimaryReferenceflag (maximum length: 1) yfs_validation_reference.is_primary_reference
The flag indicates whether or not the Reference is used to define the level at which the framework is invoked for the associated Validation.
ReferenceNamestring (maximum length: 40) yfs_validation_reference.reference_name
Indicates the reference name for identifying the approval rule violation.
