PaymentRule
PaymentRule Associated With Table: YFS_PAYMENT_RULE
This table stores all payment rules used by various organizations in the system.
AttributeTypeRequiredTable.Column
AuthRequiredBeforeScheduleflag (maximum length: 1) yfs_payment_rule.auth_required_before_schedule
This flag enables or disables the requirement of acquiring an authorization before scheduling an order.
CollectExternalThroughArflag (maximum length: 1) yfs_payment_rule.collect_external_through_ar
This flag indicates whether the collection is done externally through account receivable.
CustomerAccountMaintainedInternallyflag (maximum length: 1) yfs_payment_rule.customer_account_maintained
This flag indicates if customer accounts spending limit is established and are maintained internally. The default value is N which means customer accounts are not setup.
DefaultFlagflag (maximum length: 1) yfs_payment_rule.default_flag
This flag indicates whether a payment rule is the default rule for an organization.
Descriptionstring (maximum length: 100) yfs_payment_rule.description
Description of the Payment Rule.
FinancialHandlingRequiredflag (maximum length: 1) yfs_payment_rule.financial_handling_required
This flag indicates whether financial handling is required.
IgnorePaymentStatusForPurgeflag (maximum length: 1) yfs_payment_rule.ignore_status_for_purge
If Y, allows the order to be purged regardless of the PaymentStatus on the order. This flag is only read by the purge agent if Settlement is not required.
InterfaceTimestring (maximum length: 40) yfs_payment_rule.interface_time
This field indicates when the invoice will be published.
Allowed ValueDescription
At Creation Indicates that the invoice will be published at invoice creation time. 
At Collection Indicates that the invoice will be published at collection time, if the COLLECT_EXTERNAL_THROUGH_AR flag is set to 'N'.
