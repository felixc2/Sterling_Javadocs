ExceptionHoldflag (maximum length: 1) yfs_task.exception_hold
This flag indicates whether an exception has been detected. The exception may be system detected or user detected. Y: Exception has been detected. Exception hold reason will give the details of why the task is held. N: No prevailing exception. Once the exception is resolved, it is set to N. This attribute is used to hold the task if inventory shortage is detected by system. It is also used when supervisors want to hold tasks for operational reasons.
FinishNoLaterThandateTimeyfs_task.finish_no_later_than

The reason for the hold, if the task is held due to an exception. The system will ship with one system defined value for this field: Y01: Inventory Shortage. Users can add more reasons during installation. Example values are: 02: Equipment malfunction 03: Location Repair.
