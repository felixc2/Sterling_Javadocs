DemandLocationIdstring (maximum length: 40) yfs_activity_demand.demand_location_id
This field specifies the location identifier. This in conjunction with NODE_KEY identifies a unique location in the node, the location where an activity can be performed on the product. Typically, this field will be selected when the product should be placed in a specific location. For example, if the warehouse has just one QC location, then the quantity being requested should be placed in the QC location.

This flag indicates whether the demand is of a permanent nature. All permanent demands are never marked satisfied but purged when it is out of context. For e.g. when a shipment gets purged, all unsatisfied demands for the shipment are also purged.
