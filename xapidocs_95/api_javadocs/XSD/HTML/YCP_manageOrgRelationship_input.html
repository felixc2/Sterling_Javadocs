OrgRelationship Associated With Table: YFS_ORG_RELATIONSHIP
This table contains specific instances of relationships between the nodes.
AttributeTypeRequiredTable.Column
FromOrgCodestring (maximum length: 24) Yesyfs_org_relationship.from_org_code
The unique identifier for organization which is used as 'From' direction in the relationship.
This attribute corresponds to the OrganizationCode Fact.
Operationstring (maximum length: 40) .
This attribute specifies the kind of operation to be done.
Valid Values:
Create - To create a Relationship
Modify - Change values in an existing record. Currently not used.
Delete - To delete a Relationship
Default: If not passed then API will try to either create or modify depending on the existence of the data.

RelationshipTypeIDstring (maximum length: 40) Yesyfs_org_relationship.relationship_type_id
The unique identifier for the organization relationship type.
ToOrgCodestring (maximum length: 24) Yesyfs_org_relationship.to_org_code
The unique identifier for organization which is used as 'To' direction in the relationship.
This attribute corresponds to the OrganizationCode Fact.
ValidateOrgCodestring.
This attribute specifies whether the values passed for ToOrgCode, FromOrgCode and RelationshipTypeID need to be validated.
Valid Values: Y or N
Default: If not passed then API will not validate the values passed for ToOrgCode, FromOrgCode and RelationshipTypeID.
