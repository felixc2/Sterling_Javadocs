Location Associated With Table: YFS_LOCATION
Warehouse zones are further divided into locations of different sizes where inventory is stored. Each location belongs to a zone and a node. It is classified as INTRANSIT, REGULAR, VIRTUAL, STAGING or DOCK. It has attributes like aisle number, bay number, level number etc.
AttributeTypeRequiredTable.Column
AisleNumberinteger (total number of digits: 9) yfs_location.aisle_number
This field indicates the aisle Number of the location. The locations belong to zones which have travel aisle's between them. A zone could belong to multiple aisles and multiple zones could belong to an aisle. But a location in a zone always belongs to one and only one aisle.
BayNumberinteger (total number of digits: 9) yfs_location.bay_number
This field indicates the location's bay number. Typically, the aisle, level and bay put together represents the physical location of the location in a node, if they are based on coordinate system. The bay attribute of the location (x-coordinate from the beginning of the aisle) is used in arriving at locations nearest to the dedicated locations algorithm that is used in putaway. Typically, the bay attribute is contained within the location ID.
FreezeMoveInflag (maximum length: 1) yfs_location.freeze_move_in
This flag indicates whether the inventory is moved into the location. This is used for warehouses with certain number of locations. However, the usage of locations is phased out over a period of time.
FreezeMoveOutflag (maximum length: 1) yfs_location.freeze_move_out
This flag indicates whether inventory is moved out of the location.
InStagingLocationIdstring (maximum length: 40) yfs_location.in_staging_location_id
This field indicates that the inventory is dropped into the drop off locations.
LevelNumberinteger (total number of digits: 9) yfs_location.level_number
This field indicates the level number of the location. The height of the location (y-co-ordinate of the location from the floor) is classified as levels. The level attribute of the location is used in arriving at locations nearest to the dedicated locations algorithm used during the putaway process. Typically, the level attribute is contained within the location ID.
LocationBarCodestring (maximum length: 40) yfs_location.location_bar_code
This field indicates that a numeric barcode of the location identifier is stored for this location. The way this is intended to be used in WMS is that an RF operator when prompted to scan the location Id is also expected to scan the location bar code on the RF terminal. This ensures that the operator has traveled to the location suggested by the system and picked up or dropped off inventory at the specified location. Simply keying in the human readable form of the location ID on the RF terminal will be rejected by the system. The objective of this feature is to ensure inventory accuracy.
LocationIdstring (maximum length: 40) yfs_location.location_id
This field indicates the unique identifier for the location. The combination of the LOCATION_ID and NODE_KEY forms the unique key for the YFS_LOCATION table.
LocationKeykey (maximum length: 24) yfs_location.location_key
The primary key for the YFS_LOCATION table.
LocationSizeCodestring (maximum length: 40) yfs_location.location_size_code
This field defines the capacity of the location. All locations having the same size (dimensions and ability to hold the same weight) are classified under the same size code. This maps to the primary key attribute of the YFS_LOCATION_SIZE table.
LocationTypestring (maximum length: 40) yfs_location.location_type
This field is a system defined classification of location to aid association of locations of certain types for certain other operations with WMS. The supported types are: INTRANSIT (Mobile locations), STAGING, VIRTUAL, REGULAR and DOCK. For example, all equipment locations should be of type INTRANSIT.
Allowed ValueDescription

STAGING Staging Location 
VIRTUAL Virtual Location 
REGULAR Regular Location 
DOCK Dock Location 

MoveInSeqNointeger (total number of digits: 9) yfs_location.move_in_seq_no
This field is used by task management to suggest location while moving in the inventory (putaway). The putaway location selection algorithm uses this information to select locations from a list of locations based on the move in sequence.
MoveOutSeqNointeger (total number of digits: 9) yfs_location.move_out_seq_no
This field is used by task management to suggest location while moving out inventory (picking). The pick location selection algorithm uses this information to select locations from a list of locations based on its move in sequence.
Nodekey (maximum length: 24) yfs_location.node_key
The primary key for the YFS_LOCATION table.
OutStagingLocationIdstring (maximum length: 40) yfs_location.out_staging_location_id
This field indicates the Out Drop off Location where moves originated at this location, and may be dropped.
StorageCodestring (maximum length: 40) yfs_location.storage_code
This field is an attribute of the location that allows the warehouse to store items with similar storage profile as that of the location. For example, hazardous inflammable items need locations close to fire extinguishers. In this case the locations will be marked as having a storage code, which is suitable for storing Inflammable items. This ensures that only inflammable items get to these locations.
VelocityCodestring (maximum length: 40) yfs_location.velocity_code
This field indicates that the items are classified as 'A', 'B' or 'C' class items based on whether they are fast selling, not so fast selling and low selling items. These item classifications are typically followed by all enterprises to optimize certain operations such as sourcing and stocking. The reason to have locations that prefer certain velocity codes is to have locations closer to dock stocking 'A' class items, and locations furthest away from the dock stocking 'C' class items. The velocity code is a preference on the location and not a constraint. If 'A' class items fill up all locations meant for 'A' class items, then they can be moved to 'B' and later 'C'. Similarly, 'C' can go to 'B' and 'A' for lack of space in the respective locations preferred for a specific velocity code. The 'B' class items move into 'C' and then into 'A'.
ZoneIdstring (maximum length: 40) yfs_location.zone_id
This field indicates the zone to which a location belongs. This when associated with the node key identifies a unique zone within the node.
