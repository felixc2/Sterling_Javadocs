ActivityDemand Associated With Table: YFS_ACTIVITY_DEMAND
Demand on inventory may be placed for an activity from antother activity group. This entity stores information about such demands placed. Following attributes together form the Unique key of the table: Node, FromActivityGroup, ForActivityCode, ShipmentKey, WorkOrderKey, ForShipmentLineKey, ForWorkOrderKey, ForMoveRequestLineKey, LPNNo, LPNType, OrganizationCode, ItemId, UnitOfMeasure, ProductClass, InventoryStatus, SegmentType, Segment, DemandLocationId, OrderHeaderKey, OrderNo, DocumentType. Unique Keys of the YFS_Activity_Demand table should not be violated.

Validation ConditionError
activity demand being created should not already be existing in the database.
YCP0051Record Already Exists.
AttributeTypeRequiredTable.Column
CaseIdstring (maximum length: 40) .
CaseId against which activity demand has been placed. EnterpriseCode, ItemId, UnitOfMeasure and ProductClass define a SKU. Pallet/Case are LPN types and PalletId/CaseId represents an LPN. Either of SKU details or (PalletId/CaseId) must be passed.

Validation ConditionError
Either of (EnterpriseCode, ItemId, UnitOfMeasure and ProductClass) or (PalletId/CaseId) should be passed.
WMS00051Either of CaseId/PalletId or SKU Details must be passed.

DemandLocationIdstring (maximum length: 40) yfs_activity_demand.demand_location_id
Location Identifier. The location Id to which the demanded product must be diverted. If passed, then during put away, location considered for the demand fulfillment would be this location.

DemandQtydecimal (minimum value: 0
total number of digits: 14
fraction digits: 4) 		yfs_activity_demand.demand_qty
Quantity of the product that is demanded for the activity. If demand quantity is not passed then demand is considered to be infinite ans this demand never gets completed. For a demand without inbound shipment/order information, quantity is mandatory.
