ElementsTransaction
Transaction Associated With Table: YFS_TRANSACTION
This table stores transactions defined within the system.
AttributeTypeRequiredTable.Column
AgentJavaClassstring (maximum length: 128) yfs_transaction.agent_java_class
The java class used for consuming work flow messages being sent to a pipeline.
BaseTransactionKeykey (maximum length: 24) yfs_transaction.base_transaction_key
The base transaction that is associated with this transaction.
ChainedDocumentTypestring (maximum length: 24) yfs_transaction.chained_document_type
The chained document type.
DerivedDocumentTypestring (maximum length: 24) yfs_transaction.derived_document_type
The derived document type.
DerivedFromAbstractflag (maximum length: 1) yfs_transaction.derived_from_abstract
This flag indicates whether the transaction is derived from an abstract transaction.
DropStatusFilterstring (maximum length: 5) yfs_transaction.drop_status_filter
This field indicates how drop statuses will be filtered.
Allowed ValueDescription
ALL Indicates that derived transactions can have any drop statuses. 
EXACT Indiates that derived transactions can only have base drop statuses. 
EXTN Indicates that derived transactions can have base drop statuses or any of its extensions. 

ExternallyTriggerableflag (maximum length: 1) yfs_transaction.externally_triggerable
This flag indicates whether this transaction is externally triggerable.
ExternallyTriggeredflag (maximum length: 1) yfs_transaction.externally_triggered
This flag indicates whether the transaction is externally triggered.
IsManualflag (maximum length: 1) yfs_transaction.is_manual
This flag indicates whether this is a manual transaction requiring human intervention.
ListenerTypestring (maximum length: 2) yfs_transaction.listener_type
This field indicates the type of the listener transaction. Each process type to process type combination consistutes a different listener type.
Allowed ValueDescription
01 Indicates the chained order to sales order listener type. 
02 Indicates the derived order to sales order listener type. 
03 Indicates the shipment to order listener type. 
04 Indicates the load to shipment listener type. 
05 Indicates the receipt to shipment listener type. 
06 Indicates the receipt to order listener type. 
07 Indicates the work order to container listener type. 
08 Indicates the work order to order listener type. 

OwnerKeykey (maximum length: 24) yfs_transaction.owner_key
The owner organization that is associated with this transaction.
PickStatusFilterstring (maximum length: 5) yfs_transaction.pick_status_filter
This field indicates how pickup statuses will be filtered.
Allowed ValueDescription
ALL Indicates that derived transactions can have any pickup statuses. 
EXACT Indicates that derived transactions can only have base pickup statuses. 
BASE Indicates that derived transactions can only have base pickup statuses. 

PreDefinedflag (maximum length: 1) yfs_transaction.pre_defined
This flag indicates whether the transaction is a predefined transaction.
ProcessTypeKeykey (maximum length: 24) yfs_transaction.process_type_key
The process type that is associated with this transaction.
RequiresChainedDocTypeflag (maximum length: 1) yfs_transaction.requires_chained_doc_type
This flag indicates whether this transaction requires a chained document type.
RequiresDerivedDocTypestring (maximum length: 1) yfs_transaction.requires_derived_doc_type
This field indicates whether derived document types are required.
Allowed ValueDescription
N Indicates that it is not required. 
S Indicates a single instance of each DERIVED_DOCUMENT_TYPE for a process type and base transaction. 
M Indicates multiple instances of each DERIVED_DOCUMENT_TYPE for a process type and base transaction. 

SingleDropStatusflag (maximum length: 1) yfs_transaction.single_drop_status
This flag indicates whether this transaction can only have one drop status.
SpawnedProcessTypeKeykey (maximum length: 24) yfs_transaction.spawned_process_type_key
The process type that this transaction spawns.
SpawnsPipelineflag (maximum length: 1) yfs_transaction.spawns_pipeline
This flag indicates whether a new type of process is spawned by this transaction.
TaskQKeyTypestring (maximum length: 40) yfs_transaction.task_q_key_type
The type of tasks this transaction processes. This field is used by the workflow engine to determine if tasks needs be created for this transaction upon completion by other transaction.
TimeTriggerableflag (maximum length: 1) yfs_transaction.time_triggerable
This flag indicates whether this transaction is time triggerable.
TimeTriggeredflag (maximum length: 1) yfs_transaction.time_triggered
This flag indicates whether the transaction is time triggered.
TranSequencestring (maximum length: 15) yfs_transaction.tran_sequence
This field is not currently used.
Trangroupstring (maximum length: 20) yfs_transaction.trangroup
The group identifier for the transaction. This field can be used to better organize the transactions.
Tranidstring (maximum length: 40) yfs_transaction.tranid
The unique identifier for a transaction.
Trannamestring (maximum length: 50) yfs_transaction.tranname
The name of the transaction.
TransactionKeykey (maximum length: 24) yfs_transaction.transaction_key
The primary key for the YFS_TRANSACTION table.
Trantypestring (maximum length: 15) yfs_transaction.trantype
This field indicates the type of the transaction.
Allowed ValueDescription
SYSTEM Indicates that the transaction is system-defined. 
USER Indicates that the transaction is user-defined. 

UserTaskFlowKeykey (maximum length: 24) yfs_transaction.user_task_flow_key
The identifier of the user task flow associated with a transaction.
UserTriggerableflag (maximum length: 1) yfs_transaction.user_triggerable
This flag indicates whether this transaction is user-triggerable.
WorksOffTaskQflag (maximum length: 1) yfs_transaction.works_off_task_q
this flag indicates whether a task queue record needs to be created for this transaction. Tasks are created if a document reaches the status picked by this transaction.
