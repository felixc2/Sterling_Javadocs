ExceptionType Associated With Table: YFS_EXCEPTION_TYPE
This table houses all information about a particular exception type. ExceptionTypeKey or ExceptionType has to pass in Modify or Delete mode and in Create mode both ExceptionType and ExceptionTypeDescription have to be passed.

Validation ConditionError
If ExceptionType or ExceptionTypeDescription is not passed in Create mode.
YCP0290Both ExceptionType and ExceptionTypeDescription are mandatory.
If non-number argument is passed for Priority, ExpirationDays or FollowUpHours.
YFS10322YFS: Invalid Number
If an invalid combination of QueueId and QueueOwnerKey is passed.
YCP0326Queue cannot be found.
If existing ExceptionTypeKey or ExceptionType is passed in Create mode
YFC0001Record already exists in the database.
If record does not exist in the database
YFC0002Record does not exist in the database.
If ExceptionTypeKey or ExceptionType is not passed
YFC0300Either ExceptionTypeKey or ExceptionType is mandatory.
AttributeTypeRequiredTable.Column
Actionstring (maximum length: 40) .
Action to take
Valid Values:
Manage - Creates a new record if it does not exist or modifies an existing record if it does.
Create - Creates a new record.
Modify - Changes the value of an existing record.
Delete - Deletes an existing record.
The default behavior is Manage .

Specifies if consolidation is required for this exception type.
ConsolidationWindowstring (maximum length: 20) yfs_exception_type.consolidation_window
The consolidation window to be used for consolidation.

Validation ConditionError
If invalid consolidation window is passed.
YCP0241Invalid Consolidation Window.

Allowed ValueDescription
DAY exceptions a day old will be looked for consolidation. 
HOUR exceptions an hour old will be looked for consolidation. 
FOREVER all exceptions irrespective of time of creation will be looked for consolidation.
