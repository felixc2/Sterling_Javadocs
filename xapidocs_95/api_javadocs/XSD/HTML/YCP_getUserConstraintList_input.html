UserTranConstraint Associated With Table: YFS_USER_TRAN_CONSTRAINTS
This table stores transactional constraints, such as order number and shipment number. Transactional constraints are temporary in nature, and are automatically purged by the system when the corresponding transaction data is purged. When tasks are suggested to a user, these constraints have to be met. If no constraints are specified here, it means that any type of task can be suggested to the user.

UserZone Associated With Table: YFS_USER_ZONE
This table stores the work area definition for each user group. Work area is the set of locations, consisting of the zone and aisle combination within which a set of users work. The system suggests tasks originating and terminating only in these areas.
AttributeTypeRequiredTable.Column

This attribute corresponds to the OrganizationCode Fact.
RecordTypestring (maximum length: 2) yfs_user_zone.record_type
This field indicates whether the work area record applies to the source or the target locations zone and aisle combination. Valid values are: Option 01: Indicates that the work area record applies to the Source Zone and Aisle. Option 02: Indicates that the work area record applies to the Target Zone and Aisle.
