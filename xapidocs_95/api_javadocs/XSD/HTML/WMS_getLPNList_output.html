This flah indicates whether the LPN had variance detected during a Count operation. This drives whether the LPN can be allocated during Pick Location Assignment.
Heightdecimal (minimum value: 0
total number of digits: 14
fraction digits: 4) 		yfs_lpn_hdr.height
The height of the LPN. This will be present only if the user has overridden the dimension at LPN creation, typically during receipts. Otherwise, the dimensions are picked up based on the standard packaging type for the item.

OrganizationCodestring (maximum length: 24) yfs_lpn_hdr.organization_code
The code of the organization to which the LPN belongs. This is usually filled in situations where the LPN does not have contents. Usually, if we know the items on the LPN, we know the organization to which it belongs.

ReceiptHeaderKeykey (maximum length: 24) yfs_lpn_hdr.receipt_header_key
The receipt header key that is associated with this LPN header. This helps identify all stocks attached to a receipt. This number is of significance until the sample from the receipt has been certified as having passed or failed QC. This number is used to pull out all stocks based on the receipt. This is usually flled in situations where the LPN does not have contents.
Segmentstring (maximum length: 40) yfs_lpn_hdr.segment
The segment to which the inventory belongs. This helps to track inventory separately when the inventory is customized for a specific order so that the inventory can be allocated to that order.
SegmentTypestring (maximum length: 40) yfs_lpn_hdr.segment_type
The segment type of the inventory. This field indicates the buyer or buyer category for whom the inventory is customized.
