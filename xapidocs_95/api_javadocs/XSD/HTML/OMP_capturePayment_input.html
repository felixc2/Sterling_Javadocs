SecureAuthenticationCodestring (maximum length: 2000) .
The SecureAuthenticationCode is the code used to identify if the payment method is in the customer's possession, for example, the CVV2 code on a credit card. The knowledge of this code validates that the transaction has been made by the credit card's holder. The SecureAuthenticationCode is only used within this API call and will be discarded at the end of the API; the value will not be persisted in the database.

Operationstring (maximum length: 20) .
The operation that will be executed on the PaymentMethod. By default, the operation will be Manage. If the PaymentMethod is suspended by this API call, the MaxChargeLimit will not be set, and the ChargeSequence will be computed based upon the original or default MaxChargeLimit.
Allowed ValueDescription
Manage Creates a new record if a matching record doesn't exist. If it does exist, it modifies the matching record. 
Suspend Operates the same as Manage, except that the PaymentMethod will be suspend for charging. 
SuspendRefund Operates the same as Manage, except that the PaymentMethod will be suspend for both charges and refunds. 
Delete Deletes the matching record, if possible. If not possible, then the PaymentMethod will be suspended. 
Collect Operates the same as Manage, but additionally executes the RequestedAmount against the payment user exits. This Operation will execute partially in a separate transaction boundary and cannot be used with any of the order modification APIs or any APIs that modify orders.
