AttributeTypeRequiredTable.Column
PageNumberinteger (total number of digits: 9) .
Page number to retrieve. Page numbers are 1-based (the first page number is 1).
PageSetTokenstring.
Reference to an existing page result to extract data from. If it doesn't exist, this token will be used to identify a new set of pages. This attribute is only used when the PaginationStrategy is GENERIC.
PageSizeinteger (total number of digits: 9) .
Number of records per page
PaginationStrategystring.

Allowed ValueDescription
GENERIC Page data from any List API by executing the API normally and post-processing the results. 
ROWNUM Uses database-specific SQL commands to fetch a specific page of data. Note this is not supported on all databases. 
RESULTSET Retrieves data from the database up to and including the requested page, then discards the unused portion. 
NEXTPAGE Given a reference to the last record of a page of data, returns the next page by manipulating the SQL where clause. 

Refreshstring.

Allowed ValueDescription
Y Discard any results associated with the PageSetToken and re-execute the API. 
N Return data from a cache if available 

StartRowNumberinteger (total number of digits: 9) .
Row number of the first record in the page. This StartRowNumber takes precedence over the PageNumber attribute when determining how to retrieve records. Row numbers are 0-based (the first row number is 0).
XMLPathstring.
Path of the parent output XML element whose children will be paged. For example, the User elements in the following XML will be paged when an XMLPath of /UserList is given.
<Organization>
    <UserList>
        <User Loginid="User1"/>
        <User Loginid="User2"/>
    </UserList>
</Organization>
Note if the XMLPath is not found in the output, the output of getPage will be marked as invalid.

This attribute is only supported for the GENERIC PaginationStrategy. The default behavior is to select the root element. To explicitly select the root element, use a single slash (/).
