PaymentMethod Associated With Table: YFS_PAYMENT
PaymentMethod can be added, modified or removed to an OrderLine by ChangeOrder.
If payment processing is not required for the order, any payment method passed in the input xml is still subject to validations.
Only the PaymentMethod attributes specified in the input xml can be changed. Attempting to change other attributes on PaymentMethod, such as TotalAuthorized or TotalCharged results in an error.

Validation ConditionError
Pass changed value for attributes not specfied in input xml.
YFS10458Payment attributes cannot be modified.
AttributeTypeRequiredTable.Column
Actionstring (maximum length: 40) .
Indicates the action to take on the PaymentMethod. Valid Value: Only valid value is REMOVE .
For the addition or modification of PaymentMethod, Action does not have to be passed; if passed, its value should be Null or any value other than REMOVE . When a PaymentMethod is removed, its associated ChargeTransaction records and CreditCardTransaction records are also removed.

Validation ConditionError
Action is REMOVE , and any of the following is not zero: PendingAuthAmount, PendingChargeAmount, TotalAuthorized, TotalCharged,RequestedAuthAmount or RequestedChargeAmount.
YFS10490PaymentMethod cannot be removed
