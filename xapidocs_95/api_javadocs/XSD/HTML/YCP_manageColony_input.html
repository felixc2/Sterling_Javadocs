Colony Associated With Table: PLT_DB_COLONY
This element refers to header-level information for colony configuration. The ColonyId and PkPrefix are required during a create operation. Either is sufficient do identify the colony during a delete or modify operation.

Validation ConditionError
During a modify operation, if both ColonyId and PkPrefix are passed, the PkPrefix must match the existing record.
YFS10415Key fields cannot be modified.
The default colony (with PkPrefix of 20) cannot be deleted.
PLT0504Cannot delete the default colony.
Any resources mapped to a colony must be deleted before the colony can be deleted.
PLT0505Cannot delete the colony because colony map records still refer to it.
AttributeTypeRequiredTable.Column
Actionstring (maximum length: 40) .
Action to take
Valid Values:
Manage - Creates a new record if it does not exist or modifies an existing record if it does.
Create - Creates a new record.
Modify - Changes the value of an existing record.
Delete - Deletes an existing record.
The default behavior is Manage .

ColonyIdstring (maximum length: 40) plt_db_colony.colony_id
Unique identifier of a Colony
This attribute corresponds to the ColonyId Fact.
ColonyVersionstring (maximum length: 40) plt_db_colony.colony_version
Version of this colony, for tracking purposes. ColonyVersion will be set to the current version if it is either blank or not passed in the input and either of ColonyId or PKPrefix is passed.

PkPrefixstring (maximum length: 2) plt_db_colony.pk_prefix
Two-digit prefix for creation of table keys within the colony.

Validation ConditionError
The prefix must be between 10 and 19, inclusive, and cannot be 19.
PLT0503Invalid Primary Key Prefix

Statusstring (maximum length: 20) plt_db_colony.status
Enabled status of the colony. A colony is only enabled if the status is not set. If a status is set, then any attempt to use the colony will result in an exception (PLT0528), with this status provided as the "ErrorRelatedMoreInfo" attribute. To re-enable a colony, set the status to an empty string (""). WARNING: Disabling the default colony may render the system inoperable.

UseDefaultConfigurationstring.
Controls whether to share CONFIGURATION and STATISTICS table types with the default colony. A value of "true" is suggested for an easier and more consistent configuration, though false is the default value.
Allowed ValueDescription
false (default) Do not create any shard mappings apart from what is defined in the ColonyPool elements. 
true Copy common shard mappings from the default colony.
