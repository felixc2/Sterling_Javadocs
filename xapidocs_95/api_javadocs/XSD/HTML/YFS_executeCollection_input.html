ChargeTransactionKeykey (maximum length: 24) yfs_charge_transaction.charge_transaction_key
The primary key of the charge transaction record to process. If this key is passed, then only the corresponding charge transaction record is processed. The output document will contain AuthorizationExpirationDate, AuthorizationID, ProcessedAmount and FailedAmount.

Validation ConditionError
The ChargeTransactionKey should be for a valid charge transaction record. The record must have a status of OPEN and the ChargeType for this record should be AUTHORIZATION or CHARGE .

This attribute corresponds to the PrimaryKey Fact.
DocumentTypestring (maximum length: 40) yfs_order_header.document_type
To process all the change transaction records of the Order which have a status of OPEN , and have the ChargeType value set to AUTHORIZATION or CHARGE , pass either OrderHeaderKey or EnterpriseCode, DocumentType and OrderNo. Do not pass ChargeTransactionKey.

Passing ChargeTransactionKey indicates that only the particular charge transaction record specified should be processed.

This attribute corresponds to the OrganizationCode Fact.
IgnoreCollectionDateflag (maximum length: 1) .
If passed as N , a charge transaction record is processed only if its CollectionDate is less than or equal to current date. If passed as Y , then the CollectionDate is not considered.
Default: N
Valid Values: Y or N

IgnoreTransactionDependenciesstring.
Flag to override any transaction dependencies that may be defined for the transaction this API invokes.
Default: N
Valid Values: Y or N . When passed as Y , transaction dependencies will not be evaluated, and thus will not prevent the API from processing.
