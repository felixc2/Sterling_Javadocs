MoveRequest Associated With Table: YFS_MOVE_REQUEST
This table stores all the move requests that are created for a node. The move requests may be created from any activity group.
AttributeTypeRequiredTable.Column
FinishNoLaterThandateTimeyfs_move_request.finish_no_later_than
The timestamp for the time by which the task has to finish. This is a pass through field which is stamped on each task.
ForActivityCodestring (maximum length: 40) yfs_move_request.for_activity_code
This code identifies the activity for which the move request is being logged.
FromActivityGroupstring (maximum length: 40) yfs_move_request.from_activity_group
The department requesting the move. Valid values are: Replenishment, Outbound Picking, Receipt, Count, Value Added Services, Packing, Outbound Inventory Container Build, Shipping Container Build, and Inspection.

REPLENISHMENT Replenishment

SHIP Shipping 
INSPECTION Inspection 

HasExceptionsflag (maximum length: 1) yfs_move_request.has_exceptions

The primary key for the YFS_MOVE_REQUEST table.
MoveRequestNostring (maximum length: 40) yfs_move_request.move_request_no
The number of the move request. This field identifies a move request for a node, within the context of a module.
Nodekey (maximum length: 24) yfs_move_request.node_key
The node key that is associated with this move request.
PrioritynonNegativeInteger (total number of digits: 10) yfs_move_request.priority
An integer priority. Requests will be processed in the order of this priority.
ReasonCodestring (maximum length: 40) yfs_move_request.reason_code
This code identifies the reason why the move request was cancelled. The user defined Common Code Type is "MOVE_REQ_CANCEL_RSN".
ReasonTextstring (maximum length: 255) yfs_move_request.reason_text
The description of the reason why the move request was cancelled.
RequestUserIdstring (maximum length: 50) yfs_move_request.request_user_id
The user who requested the move.
ShipmentKeykey (maximum length: 24) yfs_move_request.shipment_key
The shipment key that is associated with this move request.
StartNoEarlierThandateTimeyfs_move_request.start_no_earlier_than
The timestamp for the time when the task can start. This is a pass through field which is stamped on each task.
Statusstring (maximum length: 40) yfs_move_request.status
The status of the move request. The system defined Common Code Type is "MOVE_REQUEST_STATUS." Valid values are: Created, Released, Cancellation Requested, Cancellation Failed and Closed.
