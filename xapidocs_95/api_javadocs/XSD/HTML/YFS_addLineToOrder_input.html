DerivedFrom Associated With Table: YFS_ORDER_LINE
Use this element to derive a new OrderLine from an existing OrderLine.
The existing OrderLine can be identified by using one of the following combinations:
-OrderLineKey
-OrderHeaderKey, PrimeLineNo, and SubLineNo
-OrderNo, EnterpriseCode, DocumentType, PrimeLineNo, and SubLineNo

If the parent line cannot be identified, an error is thrown.

To select only the part of the existing OrderLine associated with a particular release, to derive a new OrderLine from, additional data is needed. In addition to the information to identify the existing OrderLine, supply either the OrderReleaseKey or the ReleaseNo that identifies the release.

Information is taken from the parent line, based on the DerivedOrderDefaultTemplate, stored in the YFS_DOCUMENT_TEMPLATE table. The DerivedOrderDefaultTemplate is chosen on the basis of the DocumentType of the child order.

If there are attributes passed in the createOrder input xml, the values in the passed attributes are used creating the new OrderLine.
Schedule information is never propagated from the parent line to the child line.

The DerivedFromOrderLineKey, DerivedFromOrderHeaderKey and, if applicable, DerivedFromOrderReleaseKey are set on the new OrderLine. The new OrderLineis validated, like all the other lines of the new order.
A transaction is triggered. The transaction is based on the existing order's ProcessType, and the document type of the new order. This IncludeInDerivedOrder transaction changes the status of that quantity of the existing line which is equal to the OrderedQty of the child line. If the parent line does not have sufficient quantity to derive from, error is thrown.
