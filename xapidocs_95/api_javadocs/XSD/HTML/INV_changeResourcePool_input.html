AdditionalCapacityLimit Associated With Table: YFS_RES_POOL_SECONDARY_UOM
This table contains units of measures that are used as secondary resource pool capacity constraints.
AttributeTypeRequiredTable.Column
CapacityUnitOfMeasurestring (maximum length: 24) yfs_res_pool_secondary_uom.uom
The secondary capacity unit of measure in which the resource pool's capacity is limited.

Validation ConditionError
If additional capacity limits are to be defined, CapacityUnitOfMeasure attribute is mandatory. If not:
INV80_010Mandatory parameter missing.
The passed CapacityUnitOfMeasure attribute should be defined in the UOM Master table. If not:
YFS10397Invalid UOM.
The passed CapacityUnitOfMeasure attribute's UOM Type must either be WEIGHT or VOLUME. If not:
INV80_011The passed CapacityUOM must be of type WEIGHT or VOLUME only.
The passed CapacityUnitOfMeasure attribute's UOM Type should be unique. If not:
INV80_012UOM type for the additional capacity limit must be unique.

AdditionalCapacityLimitList
The dimensions in which the resource pool's capacity is constrained, in addition to the resource pool's primary capacity dimension. Additional capacity limits can only be set for a resource pool whose capacity is not maintained at the resource level. Also, once set for a resource pool, additional capacity limits cannot be modified or deleted.

Validation ConditionError
If the resource pool maintains capacity at resource level, additional capacity limits should not be passed. If not:
INV80_009Resource pool cannot have additional capacity limits if it's capacity is maintained at resource level.
AttributeTypeRequiredTable.Column
CapacitiesPerSlot
For every date range defined in YFS_RES_POOL_STD_CAPCTY_PERD, this table defines the standard capacity for each working day / shift combination.

Validation ConditionError

INV75_003Resource Pool cannot have overlapping CapacityPerSlot

ResourcePool Associated With Table: YFS_RES_POOL
A resource pool represents a collection of resources for providing services. The resources represented by a resource pool should have common characteristics (such as regions serviced and skill sets). The capacity for providing services is maintained against a resource pool.
Either ResourcePoolKey or {Node and ResourcePoolId and CapacityOrganizationCode} can be used to identify a Resource Pool.

Operationstring (maximum length: 40) .
Valid value is Delete . Set Operation = " Delete " to delete the entity.

Primary key attribute of YFS_ORGANIZATION table. This will be the Organization Code of the catalog organization of Resource Pool's Capacity Organization.
This attribute corresponds to the OrganizationCode Fact.
