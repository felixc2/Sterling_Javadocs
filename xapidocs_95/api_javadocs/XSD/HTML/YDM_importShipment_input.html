Shipment Associated With Table: YFS_SHIPMENT
If the History attribute is passed as Y , the shipment will be imported into the history tables. Otherwise if the user calls the importShipment API with the History attribute passed as N , the shipment will be imported into the normal tables. Please note that there is no validation done to ensure that the shipment being imported into the history tables does not already exist in the normal tables or vice-versa. So if you were to call importShipment twice with the same input XML, but once you pass History= N and then you pass History= Y or vice-versa, there will be problems in purge and restore at a later stage.
