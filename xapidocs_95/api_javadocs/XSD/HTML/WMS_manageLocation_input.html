Location Associated With Table: YFS_LOCATION
Warehouse zones are further divided into locations of different sizes where inventory is stored. Each location belongs to a zone and a node. It is classified as INTRANSIT, REGULAR, VIRTUAL, STAGING or DOCK. It has attributes like aisle number, bay number, level number etc.
AttributeTypeRequiredTable.Column
AisleNumberinteger (total number of digits: 9) yfs_location.aisle_number
This field indicates the aisle Number of the location. The locations belong to zones which have travel aisle's between them. A zone could belong to multiple aisles and multiple zones could belong to an aisle. But a location in a zone always belongs to one and only one aisle.
AisleSuffixstring (maximum length: 40) yfs_location.aisle_suffix
this field indicates the aisle section of the location id.
BayNumberinteger (total number of digits: 9) yfs_location.bay_number
This field indicates the location's bay number. Typically, the aisle, level and bay put together represents the physical location of the location in a node, if they are based on coordinate system. The bay attribute of the location (x-coordinate from the beginning of the aisle) is used in arriving at locations nearest to the dedicated locations algorithm that is used in putaway. Typically, the bay attribute is contained within the location ID.
CheckDigitstring (maximum length: 5) yfs_location.check_digit
Check Digit is a numeric representation of Location. It is generally used by Voice Systems to verify the Location. Location Check Digit is non unique.
DockTypestring (maximum length: 10) yfs_location.dock_type
This field depicts the type of the dock location. For a non-dock location, this field is set to blank. For a dock location, it is set to INBOUND if it is used exclusively for inbound activities. Similarly, it is set to OUTBOUND if is used exclusively for outbound activities. In case, the dock location is used for both inbound and outbound activities, this attribute is set to BOTH.
FreezeMoveInflag (maximum length: 1) yfs_location.freeze_move_in
This flag indicates whether the inventory is moved into the location. This is used for warehouses with certain number of locations. However, the usage of locations is phased out over a period of time.
FreezeMoveOutflag (maximum length: 1) yfs_location.freeze_move_out
This flag indicates whether inventory is moved out of the location.

LevelNumberinteger (total number of digits: 9) yfs_location.level_number
This field indicates the level number of the location. The height of the location (y-co-ordinate of the location from the floor) is classified as levels. The level attribute of the location is used in arriving at locations nearest to the dedicated locations algorithm used during the putaway process. Typically, the level attribute is contained within the location ID.
LocationBarCodestring (maximum length: 40) yfs_location.location_bar_code
This field indicates that a numeric barcode of the location identifier is stored for this location. The way this is intended to be used in WMS is that an RF operator when prompted to scan the location Id is also expected to scan the location bar code on the RF terminal. This ensures that the operator has traveled to the location suggested by the system and picked up or dropped off inventory at the specified location. Simply keying in the human readable form of the location ID on the RF terminal will be rejected by the system. The objective of this feature is to ensure inventory accuracy.

StorageCodestring (maximum length: 40) yfs_location.storage_code
It is an attribute of the location that allows the warehouse to store items that have the same storage profile as that of the location. For example, hazardous inflammable items need locations close to fire extinguishers. In this case the locations will be marked as having a storage code, which is suitable for storing Inflammable items. This ensures that only inflammable items get to these locations.

Validation ConditionError
StorageCode passed should be a valid location storage code for Node.
WMS00100The StorageCode is invalid.

VelocityCodestring (maximum length: 40) yfs_location.velocity_code
Items are classified as A , B or C class items based on whether they are fast selling, not so fast selling and low selling item. These item classifications are typically followed by all enterprises to optimize certain operations such as sourcing and stocking. The reason we have locations preferring certain velocity codes is that, we could have locations closer to dock stocking A class items, and locations furthest away from the dock stocking C class items. Velocity code is a preference on the location and not a constraint. If A class items fill up all locations meant for A class items, then they can go in to B and then C . Similarly C can go to B and then A for lack of space in the respective locations preferred for a specific velocity code. B class items go into C and then into A . If VelocityCode is passed blank or passed unallowed values then default VelocityCode is taken Last VelocityCode in the alphabetic sequence in common code of type VELOCITY_CODE.

Validation ConditionError
VelocityCode should be one of the valid values as identified in common code of type VELOCITY_CODE.
WMS00065Invalid Velocity Code.
