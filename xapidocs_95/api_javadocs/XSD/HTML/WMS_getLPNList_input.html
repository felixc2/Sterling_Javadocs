Inventory Associated With Table: YFS_LPN_DTL
Details of license-plated inventory are stored here.
AttributeTypeRequiredTable.Column
CountryOfOriginstring (maximum length: 40) yfs_lpn_dtl.country_of_origin
The country in which the item was manufactured.
FifoNointeger (total number of digits: 9) yfs_lpn_dtl.fifo_no
A date based inventory attribute that helps understand the order in which stock arrived at the node. This is used to send out items that arrived earlier than the ones that arrived later.
InventoryStatusstring (maximum length: 10) yfs_lpn_dtl.inventory_status
The status in which inventory is sub-classified once it is received in the warehouse. For example, product class is a classification of inventory that the product comes with when it arrives at the warehouse. Typical values of product class are First Quality and Second Quality. Due to additional inventory processes within the warehouse, it is required to further classify inventory. Typical inventory status values are Normal, Awaiting Inventory Check, Inventory Held, etc.
Quantitydecimal (minimum value: 0
total number of digits: 14
fraction digits: 4) 		yfs_lpn_dtl.quantity
The number of units of the item available at this location.
Segmentstring (maximum length: 40) yfs_lpn_dtl.segment
The segment to which the inventory belongs. This helps to track inventory separately when the inventory is customized for a specific order so that the inventory can be allocated to that order.
SegmentTypestring (maximum length: 40) yfs_lpn_dtl.segment_type
The segment type of the inventory. This helps to track inventory separately when the inventory is customized for a specific order so that the inventory can be allocated to that order. When inventory is customized, it needs to be tracked separately. This field indicates the buyer or buyer category for who the inventory is customized.
ShipByDatedateyfs_lpn_dtl.ship_by_date
The date by which the item must be shipped so that it is usable. This is also referred to as the expiration date.
