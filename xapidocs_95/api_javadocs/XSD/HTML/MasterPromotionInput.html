AttributeTypeRequiredTable.Column
Order:Promotion Associated With Table: YFS_PROMOTION
Promotions that are associated to an order or any of its lines. PromotionId must be passed to identify the promotion.
AttributeTypeRequiredTable.Column
Actionstring (maximum length: 40) .
Action to perform on this promotion.
Default: Null - if not passed, promotion will be added if it doesn't exist or modified if it does exist.
Valid Values:
CREATE - Add the specified promotion to the order.
REMOVE - Removes the specified promotion from the order.

Validation ConditionError
If an award record is associated with this promotion, it may not be removed.
OMP77_10004You are attempting to delete a Promotion associated with an award on the order.
If a promotion already exists, can't add the same promotion twice.
YFC0001Record already exists in the database.

DenialReasonstring (maximum length: 200) yfs_promotion.denial_reason
If the promotion is denied, the reason for the denial.
ExclusivityIdstring (maximum length: 100) yfs_promotion.exclusivity_id
ExclusivityID is used to group set of Promotions for mutual exclusivity. Exclusivity Type settings are applicable among Pricing Rules having same ExclusvityID.
ExclusivityTypestring (maximum length: 100) yfs_promotion.exclusivity_type
Exclusivity Type of the Promotion. Valid Values are: ALL, RULE_TYPE_ITEM, RULE_TYPE_ORDER, ONE_RULE_WITHIN_ID and GLOBAL_RULE
IsExternalflag (maximum length: 1) yfs_promotion.is_external
This flag specifies whether the promotion is external.
OverrideAdjustmentValuedecimal (total number of digits: 19
fraction digits: 6) 		yfs_promotion.override_adjustment_value
For promotions, the overridden Adjustment value used in place of the configured Adjustment of the corresponding Pricing Rule Action.
OverrideMaxAdjustmentdecimal (minimum value: 0
total number of digits: 15
fraction digits: 2) 		yfs_promotion.override_max_adjustment
For Coupon promotions, the overridden maximum adjustment value for the coupon pricing rule.

PrioritynonNegativeInteger (total number of digits: 10) yfs_promotion.priority
This attribute is used to prioritize Promotion among a list of applicable Promotions with same Exclusivity settings. Higher the value of this attribute, higher the priority of Promotion, to be applied to an order/item.
PromotionAppliedflag (maximum length: 1) yfs_promotion.promotion_applied
A flag indicating if the promotion is applied. When ValidateReferences=Y the value can be 'P' meaning the promotion was partially applied.
PromotionCategorystring (maximum length: 24) yfs_promotion.promotion_category
Specifies category of external promotion. Valid values are ORDER,ITEM and SHIPPING
PromotionGroupstring (maximum length: 6) yfs_promotion.promotion_group
The group for which this promototion belongs to. Valid values: COUPON, MANUAL.
Allowed ValueDescription
COUPON Indicates that this promotion is intended for coupons. 
MANUAL Indicates that this promotion is intended for manual pricing rules. 

PromotionIdstring (maximum length: 100) yfs_promotion.promotion_id
This identifies the promotion that is added to this order. This will be used as the CouponID when determining the price of the order.
PromotionTypestring (maximum length: 100) yfs_promotion.promotion_type
The type of promotion. Like DISCOUNT, BOGO.
ValidateReferencesflag (maximum length: 1) .
For Coupon promotions, if 'Y" the references will be validated and only valid references will be used
