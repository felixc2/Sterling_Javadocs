Instruction Associated With Table: YFS_INSTRUCTION_DETAIL
Generic table used to store instructions associated with various entities in the system.
AttributeTypeRequiredTable.Column
InstructionDetailKeykey (maximum length: 24) yfs_instruction_detail.instruction_detail_key
Primary key attribute of YFS_INSTRUCTION_DETAIL table that identifies an instruction.
InstructionTextstring (maximum length: 4000) yfs_instruction_detail.instruction_text
Free-form text indicating any additional information related to the instruction type or otherwise.
InstructionTypestring (maximum length: 10) yfs_instruction_detail.instruction_type
Categorization of instructions so that they can be grouped together for reference. For example, you may have instructions of type Pick, Pack, and Ship for an order.
InstructionURLstring (maximum length: 255) yfs_instruction_detail.instruction_url
The URL relating to and/or containing the instructions.
InstructionUsagestring (maximum length: 10) yfs_instruction_detail.instruction_usage
Another way of categorization of instructions, over and above the instruction type.
ReferenceKeykey (maximum length: 24) yfs_instruction_detail.reference_key
Link to the entity key for which the instruction is stored. For example, if this record stores order header level instructions, then this attribute stores the value of order header key.
SequenceNononNegativeInteger (total number of digits: 5) yfs_instruction_detail.sequence_no
You could have an instruction exceeding the limits of a single record. Sequence number is used to provide a way to store a link between such instructions.
TableNamestring (maximum length: 40) yfs_instruction_detail.table_name
Identifies the entity for which this instruction is stored. For example, if this record stores order header instructions, then you will see this field reading YFS_ORDER_HEADER.
