ElementsCustomerAssignment
CustomerAssignment Associated With Table: YFS_CUSTOMER_ASSIGNMENT
Specify assignments of Team to Customer or a combination of Team and Customer and User or Customer to a valid user of a Buyer Organization. Team is a manadatory attribute if the assignment is not to a valid buyer user. Team is also mandated if the organization of the User has roles other than the Buyer.

DeleteChildTeamAssignmentsstring.

Whenever the operation is Delete and UserId is passed then only the relevant Customer assignment is deleted. In this case if the deletion is successful, the AssignedReps on the customer is decreased by 1. If the UserId is not passed then based on this flag the Customer assignments are deleted.
Valid Values:
Y - Indicates that assignments associated with the child Teams of the Team passed, need to be deleted.
N - Only relevant Customer and Team assignments are deleted and child Team assignments are not considered for deletion.

Operationstring (maximum length: 40) .
Action to take.
Valid Values:
Create - Creates a new record.
Delete - Deletes an existing record.
Whenever the operation is Create and UserId is passed, then validation for maximum assignments is done. The maximum number of assignments cannot be greater than the value of AssignedReps on the Customer. If the customer assignment to the team and user is successful the AssignedReps value on the Customer is increased by 1. If the Customer is assigned to a valid CustomerContact this validation is not done.

Validation ConditionError
Action is Create and UserId is passed and maximum assignments is reached.
YCP80_0004Maximum Customer Assignments Reached

OrganizationCodestring (maximum length: 24) yfs_customer_assignment.organization_code
Organization code of organization that defines this Team or User. Both the Team and User could belong to same organization or to diffrent organizations. If Team and User belong to different organizations, then the OrganizationCode should be the organization of the Team but within the same organization hierarchy of the User's organization. If TeamCode is passed then a valid OrganizationCode has to be passed. When assigning Buyer user to a Customer, this attribute should correspond to the Organization Code of the Buyer User's Organization.
