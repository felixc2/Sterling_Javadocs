Adjustment Associated With Table: YFS_INVENTORY_PENDING_ADJ
Inventory costing related table. Stores adjustments that have not been posted yet for costing purposes. Details for the items picked for adjustments. The attributes UnitCost, AccountNo, Reference_1, and Reference_2 are used only if PostType is 02 ; otherwise, the individual record's fields are used. PostTypes: 01 for Individual and 02 for summed up. 01 is the default.
AttributeTypeRequiredTable.Column
AccountNostring (maximum length: 40) yfs_inventory_pending_adj.account_no
Used only if PostType is 01 ; otherwise used from item element.

AdjustmentKeykey (maximum length: 24) Yesyfs_inventory_pending_adj.inventory_pending_adj_key
AdjustmentKey is the primary key of for the YFS_INVENTORY_PENDING_ADJ table. Adjustments are posted for this record. If AdjustmentKey is passed, it should exist in the YFS_INVENTORY_PENDING_ADJ table and the status should not be 02 . If AdjustmentKey is not passed, the status is defaulted to 01 .

Validation ConditionError
If no record is found for the key:
YFS10001No Record Found
If status for the record is 02:
YFS10401Already Posted
