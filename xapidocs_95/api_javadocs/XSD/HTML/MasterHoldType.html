HoldTypeTran Associated With Table: YFS_HOLD_TYPE_TRAN
This table stores the mapping of the hold types that should get enabled on an order when a transaction executes, as well as the mapping in between the hold types and the prevented transactions.
AttributeTypeRequiredTable.Column
HoldEffectLevelstring (maximum length: 40) yfs_hold_type_tran.hold_effect_level
Indicates whether transaction should be prevented at ORDER level or ORDER_LINE level. Default value is ORDER for Base Process Type 'ORDER_FULFILLMENT'.
HoldTypeTranKeykey (maximum length: 24) yfs_hold_type_tran.hold_type_tran_key
Primary key attribute of YFS_HOLD_TYPE_TRAN.
Purposestring (maximum length: 40) yfs_hold_type_tran.purpose
The purpose of the association in between the transaction and the hold type.
Allowed ValueDescription
APPLY When an order is executed by the transaction, the hold type will be applied to the order. This is the default value. 
PREVENT When an order is on a hold of this type, the order can not be processed by the transaction. 

TransactionIdstring (maximum length: 40) yfs_hold_type_tran.transaction_id
The transaction which can apply the hold type on an order, or the transaction which will be prevented to process an order on this hold type.
