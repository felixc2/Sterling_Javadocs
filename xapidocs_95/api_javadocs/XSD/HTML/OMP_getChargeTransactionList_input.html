ElementsChargeTransactionDetail
ChargeTransactionDetail Associated With Table: YFS_CHARGE_TRANSACTION
This table stores information about financial transactions associated with order processing. Every action carried out on an order which may have an impact on financial components is logged in this table.
AttributeTypeRequiredTable.Column
AsyncRequestIdentifierstring (maximum length: 255) yfs_charge_transaction.async_request_identifier
This field is applicable for only Asynchronous charge requests. This field uniquely identifies the charge transaction marked as asynchronous. By default the identifier is same as primary key i.e Charge Transaction Key will be set as Async Request Identifier. This identifier can be used for recording the response to the asynchronous request.
AuditTransactionIDstring (maximum length: 40) yfs_charge_transaction.audit_transaction_id
The identifier of the order modification audit that caused this record.
AuthorizationExpirationDatedateTimeyfs_charge_transaction.authorization_expiration_date
The date and time until which the authorization specified by the authorization ID is applicable.
AuthorizationIDstring (maximum length: 255) yfs_charge_transaction.authorization_id
For authorization requests, this field holds the identifier of the authorization that was made. For charge requests, this field holds the authroization ID to be used for charging the amount.
BookAmountdecimal (total number of digits: 15
fraction digits: 2) 		yfs_charge_transaction.book_amount
The amount that could be charged to the buyer if the order was fulfilled. At the time of fulfillment, a negative entry is made in this column and positive entry in the DEBIT_AMOUNT column.
CallForAuthorizationStatusstring (maximum length: 20) yfs_charge_transaction.call_for_auth_status
This column indicates the transaction is valid for a call for authorization and whether or not the authorization code was acquired.
Allowed ValueDescription
REQUIRES_CALL If a call for auth needs to be placed. 
AUTH_CODE_ACQUIRED Once the auth code has been entered. 
CLOSED Once the gateway has been communicated with and the funds have been transferred via the normal means.

ForAsyncRequestIdentifierstring (maximum length: 255) yfs_charge_transaction.for_async_request_identifier
Reference to AsyncRequestIdentifier, This field is applicable only for charge transactions that are recorded as response to an Asynchronous Charge request. This field should refer to the Async Request Identifier field of the original Charge Request that was marked as asynchronous. This field enables tracking recorded responses for asynchronous charge requests.
HoldAgainstBookflag (maximum length: 1) yfs_charge_transaction.hold_against_book
This flag indicates whether the request was created for a precharge or a payment method whose payment type has the ChargeInsteadOfAuth flag turned on.
InPersonflag (maximum length: 1) yfs_charge_transaction.in_person
This flag indicates that the transaction represents an in person, or card in hand, transaction.
OfflineStatusflag (maximum length: 1) yfs_charge_transaction.offline_status
This flag indicates the transaction attempted to reach the payment gateway within a UserExit, but the UserExit was unable to contact the gateway. This may be updated to N by the user exit, and will be reset to null when the record moves to CLOSED status.

The user exit call status of this record. The status will be "INVOKED" if the transaction fails during or after the user exit call, otherwise the value will be blank.
VoidTransactionflag (maximum length: 1) yfs_charge_transaction.void_transaction
This flag indicates that the Charge Transaction is either a Void request or submitted for a Void operation. Valid values for a Void Charge Transaction are V (is a void request), R (is a forced refund request), C (closed error record, already recreated) and null. Valid value for a Charge Transaction that has been requested for Void operation is S (void submitted for this transaction).
