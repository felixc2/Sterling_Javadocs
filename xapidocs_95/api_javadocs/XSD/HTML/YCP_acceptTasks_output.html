TaskType Associated With Table: YFS_TASK_TYPE
This table stores different types of tasks performed in a warehouse. Different task types typically represent tasks that are physically different. For instance, paper based putaway as opposed to RF based putaway.
AttributeTypeRequiredTable.Column
ActivityCodestring (maximum length: 40) yfs_task_type.activity_code
The code of the activity within the Activity Group. For example, STORAGE. This is system defined.
ActivityGroupIdstring (maximum length: 40) yfs_task_type.activity_group_id
The activities performed in a warehouse are grouped in an activity group Id. For example, PUTAWAY. Activity groups are system defined.
BatchRequiredflag (maximum length: 1) yfs_task_type.batch_required
This flag indicates whether tasks need to be batched on creation.Y: When tasks are created, they need to be batched and printed. This typically represents a paper based process, but it may also represent an RF based process. N: No batching required This information is used by createTask API: When tasks are created, if this flag is Y, then they are 'Held for batching'.
BatchRuleKeykey (maximum length: 24) yfs_task_type.batch_rule_key
The batch rule key that is associated with this task type.
ExecTasksThruDropoffflag (maximum length: 1) yfs_task_type.exec_tasks_thru_dropoff
This flag indicates whether to execute tasks through drop off locations. Y: indicates that this type of task is physically broken up by drop off locations (on system, this does not automatically translate into multiple tasks). N: indicates that this type of task is carried out without being broken up into drop-off locations. That is, product is carried directly from origin to destination without being stopped at drop off locations. This information is used by createTask API: If this is set to Y, and pick method is not 'pick to pallet', then tasks are created split by drop off location (drop off chaining). Putaway RF screen: if this is Y, then it displays the drop off location on pick screen if that drop off location is different from the target location on the task. updateTaskStatus API: when status is moved to PICKED, then if the task needs to be split, then internally, createTask API is called, which considers this rule.
HoldTaskOnCreationflag (maximum length: 1) yfs_task_type.hold_task_on_creation
This flag indicates whether tasks are held in creation. Tasks held on creation may be released manually later. This information is used by the createTask API. When tasks are created, if this flag is Y, then they are 'Held for batching'.
IsAdHocMoveflag (maximum length: 1) yfs_task_type.is_ad_hoc_move
This flag indicates whether putaway can be performed on an ad hoc basis.
MaxUsersAllowednonNegativeInteger (total number of digits: 5) yfs_task_type.max_users_allowed
The number of users who can be suggested tasks of this task type and primary task reference combination.
Nodekey (maximum length: 24) .
The node key that is associated with this task type. Since operations vary for each node, task type level attributes are maintained at a node level.
OrganizationCodestring (maximum length: 24) yfs_task_type.organization_code
The owner organization of the node for which this task type is defined.
PickIntoInventoryContainerstring (maximum length: 40) yfs_task_type.pick_into_inventory_container
The case Id or pallet Id into which products are picked directly.
PickIntoOutboundContainerstring (maximum length: 40) yfs_task_type.pick_into_outbound_container
The outbound case Id or pallet Id into which the item is picked.
PrimaryTaskReferencestring (maximum length: 100) yfs_task_type.primary_task_reference
The smallest group of tasks that need to be kept together for a particular user. For instance, in the case of strict order pick, user picks one shipment at a time. In such a case, if the first task that is suggested to a user has shipment number A, then all other tasks belonging to that shipment number must be suggested only to that user. In order to accomplish this, system automatically creates a temporary user level transaction constraint for this shipment.
ProductivityTypestring (maximum length: 40) yfs_task_type.productivity_type
The productivity type to which this Task type is associated. It is possible that a task type can not have a productivity type associated.
ResourceIdOverridestring (maximum length: 40) yfs_task_type.resource_id_override
The unique identifier of a resource, which will be used for a task type. This resource will override the default resource associated with the task type.
SourceTaskOrderBystring (maximum length: 4) yfs_task_type.source_task_order_by
The sequence in which to order the source task. 01: Move-In Sequence. Typically, this is used during putaway. 02: Move-out Sequence. Typically, this is used during replenishment and order picking.
SourceTaskSuggestionstring (maximum length: 4) yfs_task_type.source_task_suggestion
The suggestion for the source task. 01: User sequenced User is presented with a set of tasks, and the user selects the order in which they are done 02: System sequenced System presents the user with tasks one by one, and the user has to perform them in that order (override possible).
StandardRatenonNegativeInteger (total number of digits: 5) .
The rate at which tasks of this task type are executed. The rate can be either the number of tasks per hour or in units per hour, as specified in standard rate unit of measure. It is derived from allowed minutes per task column in YFS_PRODUCTIVITY_TYPE table.
StandardRateUomstring (maximum length: 4) .
The unit of measure at which tasks of this task type are executed. The rate can be either number of tasks per hour or in units per hour.
TargetTaskOrderBystring (maximum length: 4) yfs_task_type.target_task_order_by
The sequence in which to order target tasks. 01: Move-In Sequence Typically, this is used during putaway. 02: Move-out Sequence Typically, this is used during replenishment and order picking.
TargetTaskSuggestionstring (maximum length: 4) yfs_task_type.target_task_suggestion
The suggestion for the target task. 01: User sequenced - User is presented with a set of deposit tasks, and the user selects the order in which deposit is done 02: System sequenced - System presents the user with deposit tasks one by one, and the user has to deposit in that order (override possible).
TaskTypestring (maximum length: 10) yfs_task_type.task_type
