Shipment Associated With Table: YFS_SHIPMENT
The shipment for which to be unconfirmed. Pass ShipmentKey or pass ShipmentNo, ShipNode and SellerOrganizationCode to identify the shipment.

Validation ConditionError
Must be a valid shipment.
YDM00017Invalid Shipment.
Shipment cannot be unconfirmed if any one of the following conditions is true.
  • Shipment is closed.
  • Shipment has not been confirmed.
  • Shipment has been invoiced and that invoice has not been voided.
  • If the shipment is part of a container and the container is closed.
  • If the shipment is part of a manifest and the manifest is closed.
  • If the shipment is part of a load and the load has been received or is in transit.
  • If a shipment line is part of a work order and the work order is fully or partially completed.
  • If a receipt has been started for the shipment.

ShipmentStatusAudit Associated With Table: YFS_SHIPMENT_STATUS_AUDIT
This table stores shipment status change audit. Each time the shipment status changes because of a business process, an audit is posted along with the status, timestamp, reason code and text, user ID and program ID. All transactions in the shipment pipeline have entries in this table upon completion of a transaction.
