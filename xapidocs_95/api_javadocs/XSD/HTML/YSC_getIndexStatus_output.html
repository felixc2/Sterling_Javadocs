IndexConfig Associated With Table: PLT_INDEX_CONFIG
This table contains status of index
AttributeTypeRequiredTable.Column
IndexNamestring (maximum length: 25) plt_index_config.index_name
The actual name of the index if index is defined in the entity xml (non custom mode) or the index config identifer defined in a configuration xml (custom mode).
IndexTemplatestringplt_index_config.index_template
The template for the index. This is used to retrieve the data to index. In case of custom mode it holds the index definition identified the indexConfigId and for non-custom mode this contains an entity output template with all indexable attributes.
IndexUpdateRequiredflag (maximum length: 1) plt_index_config.index_update_required
This flag indicates whether the recent update is done on the record
IndexVersionnonNegativeInteger (total number of digits: 10) plt_index_config.index_version
This field indicates the version of index definition.
IndexStatus Associated With Table: YFS_INDEX_STATUS
This table contains the sanity of the index server with respect to index build and search operations. Both these operations read this table before attempting on index server. Besides that they update this table if any issues with the index server is found during the opearation.
AttributeTypeRequiredTable.Column
IndexNamestring (maximum length: 25) yfs_index_status.index_name
The name of the index. This is the XMLName of the corresponding entity. For eg: Indexing is enabled for Order_Header and Shipment entities. Hence name of the indices would be Order and Shipment respectively.
IndexWorkingflag (maximum length: 1) yfs_index_status.index_working
This indicates that indexing operation is working for this index.
NumberOfDocsstring.
The number of documents in the index.

SearchWorkingflag (maximum length: 1) yfs_index_status.search_working
This indicates that search operation is working for this index.
SizeInBytesstring.
The size of the index.

IndexSync Associated With Table: YFS_INDEX_SYNC
This table contains the information whether the index is in sync with the database. The search operation checks this table and searches on index only if it is sync. The index update and sync agents modifies this table with the sync results.
AttributeTypeRequiredTable.Column
EnterpriseCodestring (maximum length: 24) yfs_index_sync.enterprise_code
The enterprise to which the index syncing operation is tracked.
InSyncflag (maximum length: 1) yfs_index_sync.in_sync
This indicates that the index is in sync with the database.
IndexNamestring (maximum length: 25) yfs_index_sync.index_name
The name of the index. This is the XMLName of the corresponding entity. For eg: Indexing is enabled for Order_Header and Shipment entities. Hence name of the indices would be Order and Shipment respectively.
