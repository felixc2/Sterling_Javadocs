PaymentConfiguration Associated With Table: YFS_PAYMENT_CARD_TYPE
This element contains the configuration of payment rules that impact how payment processing will use authorizations and charges.
AttributeTypeRequiredTable.Column
AllowAuthorizationsToExceedSettlementRequestflag (maximum length: 1) yfs_payment_card_type.allow_auth_to_exceed_settle_rq
This rule determines whether authorizations larger than the settlement amount are able to be used for settlement for this card type. If IS_MULTIPLE_REQ_ALLOWED or CHARGE_CONSOL_ALLOWED are Y, this is implicitly Y.
AuthorizationExpirationHoursnonNegativeInteger (total number of digits: 7) yfs_payment_card_type.auth_expiration_hours
Number of hours an authorization is valid before expiring for this payment card type. This value is not used internally by the system, but may be referenced for the computation of the AuthorizationExpirationDate when processing payment transactions.
AuthorizationReversalHoursnonNegativeInteger (total number of digits: 7) yfs_payment_card_type.auth_reversal_hours
Number of hours an authorization is valid for reversal for this payment card type. This value is not used internally by the system and serves as an informational field.
AuthorizationReversalStrategystring (maximum length: 20) yfs_payment_card_type.auth_reversal_strategy
This field controls when reverse authorization requests will be created for this payment card type.
Allowed ValueDescription
NO_REVERSE This strategy will never generate reversal requests for authorizations. This is the default strategy. 
EXPIRED This strategy will generate a reversal request for an authorization after it expires. 
REVERSE_EXCESS This strategy will still reverse authorizations that are expired, but it will also reverse authorizations that are in excess of either the order total or the CTR they are mapped to. In additon, any authorization that cannot be mapped to a CTR and is marked as unusable will also be reversed.
