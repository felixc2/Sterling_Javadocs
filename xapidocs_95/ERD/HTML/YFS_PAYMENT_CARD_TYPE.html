Entity Definition
YFS_PAYMENT_CARD_TYPE
( CONFIGURATION Table )
This table stores all payment card types.



You can extend this table by adding columns but not by creating hang-off table(s). Please refer to the Customization Guide for detailed procedures and restrictions.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
PAYMENT_CARD_TYPE_KEYChar (24)The primary key for the YFS_PAYMENT_CARD_TYPE table.  
PAYMENT_CARD_TYPEVarchar2 (100)The unique identifier for the payment card type. 
ORGANIZATION_CODEChar (24)The organization that is associated with this payment card type.
A logical foreign key to the following tables:
  • YFS_ORGANIZATION
  • YFS_PAYMENT_TYPE (in combination with the PAYMENT_TYPE column) 
PAYMENT_TYPEVarchar2 (20)The unique identifier for the payment type of this payment card type. Currently, the system only supports payment types of payment type group CREDIT_CARD, DEBIT_CARD,STORED_VALUE_CARD and OTHER.
A logical foreign key to the following table:
  • YFS_PAYMENT_TYPE (in combination with the ORGANIZATION_CODE column) 
AUTH_REVERSAL_STRATEGYVarchar2 (20)This field controls when reverse authorization requests will be created for this payment card type. The valid values are:
  • NO_REVERSE: This strategy will never generate reversal requests for authorizations. This is the default strategy.
  • EXPIRED: This strategy will generate a reversal request for an authorization after it expires.
  • REVERSE_EXCESS: This strategy will still reverse authorizations that are expired, but it will also reverse authorizations that are in excess of either the order total or the CTR they are mapped to. In additon, any authorization that cannot be mapped to a CTR and is marked as unusable will also be reversed. 
CHARGE_CONSOL_ALLOWEDChar (1)This flag indicates whether charge consolidation is allowed for this payment card type. 
MIN_WAIT_TIME_FOR_CONSOLNumber (14,4)The minimum time to wait for utilizing the opportunity to consolidate charges for this payment card type. 
IS_MULTIPLE_REQ_ALLOWEDChar (1)Use Same Authorization Multiple Times. This flag controls whether the same authorization request can be allowed multiple times for this payment card type. 
ALLOW_AUTH_TO_EXCEED_SETTLE_RQChar (1)This rule determines whether authorizations larger than the settlement amount are able to be used for settlement for this card type. If IS_MULTIPLE_REQ_ALLOWED or CHARGE_CONSOL_ALLOWED are Y, this is implicitly Y. 
AUTH_EXPIRATION_HOURSNumber (7,0)Number of hours an authorization is valid before expiring for this payment card type. This value is not used internally by the system, but may be referenced for the computation of the AuthorizationExpirationDate when processing payment transactions. 
AUTH_REVERSAL_HOURSNumber (7,0)Number of hours an authorization is valid for reversal for this payment card type. This value is not used internally by the system and serves as an informational field. 
SHORT_DESCRIPTIONVarchar2 (40)The short description of the payment card type. 
LONG_DESCRIPTIONVarchar2 (100)The long description for the payment card type. 
CHARGE_UP_TO_AVAILABLEChar (1)This flag indicates whether the user exit getFundsAvailable should be invoked to determine the funds available on this payment card type. 
PARTIAL_REVERSAL_SUPPORTChar (1)This rule determines whether reverse authorization requests support partial reversal for this payment card type. If Y, partial reversal is supported. If N, only full reversal is supported. 
ZERO_AMOUNT_AUTH_SUPPORTEDChar (1)This flag indicates whether or not the system will attempt a zero authorization when this Payment Type is added on the Order. 
NO_PROCESSING_REQUIREDChar (1)This flag indicates that when the Payment Type is added on the Order, it is already processed. Examples of this Payment Type are checks/cash. A pre processed Charge Transaction record of type CHARGE is created for this Payment Type. 
VOIDABLEChar (1)This flag indicates that the Payment Type allows a void operation. 
CASH_BACK_LIMITNumber (15,2)The maximum amount that can be given back as cash for payment methods associated with this payment card type. 
CREATETSDateTimeThe timestamp for the creation of this record. 
MODIFYTSDateTimeThe timestamp for the last modification of this record. 
CREATEUSERIDVarchar2 (40)The user who created this record. 
MODIFYUSERIDVarchar2 (40)The user who last modified this record. 
CREATEPROGIDVarchar2 (40)The program that created this record. 
MODIFYPROGIDVarchar2 (40)The program that modified this record. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.
