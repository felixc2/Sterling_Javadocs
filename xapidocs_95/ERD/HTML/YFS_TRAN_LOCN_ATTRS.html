YFS_TRAN_LOCN_ATTRS
( TRANSACTION Table )
Location's transactional attributes like volume, weight, empty/full etc., are maintained in this table. The reason this is not a part of the YFS_LOCATION table is because we would like to use caching capacity for heavily accessed master configurations such as locations, items etc. Only those attributes that are likely to be modified based on transactions need to be read from the database at every access.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

LOCATION_TYPEVarchar2 (40)This field is a system defined classification of location to aid association of locations of certain types for certain other operations with WMS. The supported types are: INTRANSIT (Mobile locations), STAGING, VIRTUAL, REGULAR and DOCK. For example, all equipment locations should be of type INTRANSIT. The valid values are:
  • INTRANSIT: In Transit Location
  • STAGING: Staging Location
  • VIRTUAL: Virtual Location
  • REGULAR: Regular Location
  • DOCK: Dock Location 
ZONE_IDVarchar2 (40)This field indicates the zone to which a location belongs. This when associated with the node key identifies a unique zone within the node. 
MOVE_IN_SEQ_NONumber (9,0)This field is used by task management to suggest location while moving in the inventory (putaway). The putaway location selection algorithm uses this information to select locations from a list of locations based on the move in sequence. 
VELOCITY_CODEVarchar2 (40)This field indicates that the items are classified as 'A', 'B' or 'C' class items based on whether they are fast selling, not so fast selling and low selling items. These item classifications are typically followed by all enterprises to optimize certain operations such as sourcing and stocking. The reason to have locations that prefer certain velocity codes is to have locations closer to dock stocking 'A' class items, and locations furthest away from the dock stocking 'C' class items. The velocity code is a preference on the location and not a constraint. If 'A' class items fill up all locations meant for 'A' class items, then they can be moved to 'B' and later 'C'. Similarly, 'C' can go to 'B' and 'A' for lack of space in the respective locations preferred for a specific velocity code. The 'B' class items move into 'C' and then into 'A'. 
LOCATION_SIZE_CODEVarchar2 (40)This field defines the capacity of the location. All locations having the same size (dimensions and ability to hold the same weight) are classified under the same size code. This maps to the primary key attribute of the YFS_LOCATION_SIZE_CODE table. 
STORAGE_CODEVarchar2 (40)This field is an attribute of the location that allows the warehouse to store items with similar storage profile as that of the location. For example, hazardous inflammable items need locations close to fire extinguishers. In this case the locations will be marked as having a storage code, which is suitable for storing Inflammable items. This ensures that only inflammable items get to these locations. 
FILL_CODEChar (1)Indicates if the location is Empty('0'), Partially Full('1') or Full('2'). These three values are updated by the applications based on stock movements to and from the location. The valid values are:
  • 0: Empty
  • 1: Partially Full
  • 2: Full 
AVAILABLE_VOLUMENumber (14,4)This field indicates that the unoccupied physical volume of the location that is available for moving in stock. This does not include the PEND_IN_VOLUME. (See explanation against this field in this entity). 
AVAILABLE_WEIGHTNumber (14,4)This field indicates the additional physical weight the location can withstand when products are moved into this location to fill in the unoccupied volume. This does not include the PEND_IN_WEIGHT. (See explanation against this field in this entity). 
PEND_IN_VOLUMENumber (14,4)This field indicates the volume of the stock expected to move into this location. 
PEND_IN_WEIGHTNumber (14,4)This field indicates the weight of the stock expected to move into this location. 
NET_AVAIL_VOLUMENumber (14,4)This field indicates the volume of the location that is available after considering the PEND_IN_VOLUME. 
NET_AVAIL_WEIGHTNumber (14,4)This field indicates the weight that this location can withstand after considering the PEND_IN_WEIGHT. 
FREEZE_ON_VARIANCEChar (1)This flag indicates whether the location has an item with negative variance that was detected during the count operation. This determines whether a pick or putaway operation can be performed in the location. 
FREEZE_MOVE_INChar (1)This flag indicates whether the inventory is moved into the location. This is used for warehouses with certain number of locations. However, the usage of locations is phased out over a period of time. 
LAST_COUNTED_DATEDateTimeThis attribute records what is the date on which the location has been put on count. 
LAST_MANUALLY_PUT_ON_COUNTDateTimeThis attribute records what is the date on which the location has been manually put on count.
