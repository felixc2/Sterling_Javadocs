Entity Definition
YPM_PRICELIST_ASSIGNMENT
( MASTER Table )
This table maintains the assignment of price list to customer, based on customer identifier or attributes of a customer. A price list is assigned to a customer as long as all the values in the assignment match that of the customer.



You can extend this table by adding columns but not by creating hang-off table(s). Please refer to the Customization Guide for detailed procedures and restrictions.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
PRICELIST_ASSIGNMENT_KEYChar (24)The primary key for the YPM_PRICELIST_ASSIGNMENT table.  
ENTERPRISE_CODEChar (24)Organization code of the enterprise to which assignment belongs.
A logical foreign key to the following table:
  • YFS_ORGANIZATION 
CUSTOMER_IDChar (40)The identifier for the customer. If null then this assignment is applicable irrespective of the CustomerID of a customer. 
CUSTOMER_TYPEChar (2)Indicates the type of customer. If null then this assignment is applicable irrespective of the type of a customer. The valid values are:
  • 01: Indicates that this assignment is for business customer.
  • 02: Indicates that this assignment is for consumer customer.
  • 03: Indicates that this assignment is for anonymous customer.
  • 04: Indicates that this assignment is for everyone. 
PRICELIST_HDR_KEYChar (24)Price list that is assigned. 
SHAREABLEChar (1)Indicates whether the price list should be shared with the customer's descendant customers or not. Applicable only when CustomerID is populated on this assignment. 
VERTICALVarchar2 (40)Vertical of the customer to which the price list is assigned. If null then this assignment is applicable irrespective of the vertical of a customer. 
RELATIONSHIP_TYPEVarchar2 (40)The relationship type of the customer. If null then this assignment is applicable irrespective of the relationship type of a customer. 
CUSTOMER_LEVELVarchar2 (40)The level of a customer with the enterprise. If null then this assignment is applicable irrespective of the membership level of a customer. 
REGION_KEYChar (24)The region to which the price list is assigned. If null then this assignment is applicable irrespective of the region of a customer.
A logical foreign key to the following table:
