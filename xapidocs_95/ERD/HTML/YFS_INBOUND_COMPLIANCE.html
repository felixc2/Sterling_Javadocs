Entity Definition
YFS_INBOUND_COMPLIANCE
( CONFIGURATION Table )
This table stores the configuration details for an inbound compliance.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns

ORGANIZATION_CODEChar (24)The organization code for whom this configuration is applicable.
A logical foreign key to the following table:

IS_APPOINTMENT_REQDChar (1)This flag indicates whether this buyer organization requires a customer appointment check. The default value is 'N'. 
IS_ROUTING_GUIDE_MAINTAINEDChar (2)This field indicates whether this buyer organization maintains routing guides. Valid values can be 01 (Routing guide is maintained within the system), 02 (Routing guide is maintained external to the system) and 03 (Routing guide is not maintained). 
IS_ESP_MAINTAINEDChar (1)This flag indicates whether this buyer organization maintians ESP parameters. The default value is 'N'. 
SCM_LABEL_REQDVarchar2 (40)A field which indicates if SCM labels are required or not and if required, are lables. 
SCM_LABEL_LEVELVarchar2 (40)A field which indicates if SCM labels are to be generated at the Case level or for every Shipping Container. 
READ_UOM_FROM_BUYERChar (1)This flag specifies if the unit of measure's are to be read from the buyer's catalog or seller's catalog. 
CAPTURE_TAG_WHILE_SHIPPINGChar (1)This flag indicates whether to capture tag information during shipping. Valid values are Y and N. 
CAPTURE_SERIAL_WHILE_SHIPPINGChar (1)This flag indicates whether to capture Serial information during shipping. Valid values are Y and N.
