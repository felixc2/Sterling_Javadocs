Entity Definition
YFS_COUNT_STRATEGY
( CONFIGURATION Table )
This configuration table stores parameters used to determine the task type for a task associated with the task type group count.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
COUNT_STRATEGY_KEYChar (24)The primary key for the YFS_COUNT_STRATEGY table.  
DESCRIPTIONVarchar2 (100)A brief description of the count strategy. 
TASK_TYPEChar (10)This field is a short two-digit code that identifies the task type.
A logical foreign key to the following table:
  • YFS_TASK_TYPE (in combination with the ORGANIZATION_CODE column) 
CNT_LOC_EVEN_IF_NO_INVChar (1)This flag is applicable when strategy is defined for a zone. This flag indicates whether the count tasks are generated for all locations without verifying the inventory availability in the zone. 
ACTIVITY_GROUP_IDVarchar2 (100)The activity group system defined. This field indicates the task type that should be associated to an activity group. 
ORGANIZATION_CODEChar (24)This field indicates the organization code of the node.
A logical foreign key to the following tables:
  • YFS_TASK_TYPE (in combination with the TASK_TYPE column)
  • YFS_SHIP_NODE
  • YFS_ORG_ENTERPRISE (in combination with the ENTERPRISE_KEY column) 
TASK_GRANULARITYVarchar2 (40)This field specifies the task granularity with which the count tasks will be performed. The valid values are:
  • REQUEST: Request
  • VARIANCE: Variance 
ENTERPRISE_KEYChar (24)The enterprise key that is associated with this count strategy.
A logical foreign key to the following tables:
  • YFS_ORG_ENTERPRISE (in combination with the ORGANIZATION_CODE column)
  • YFS_ENTERPRISE 
ZONE_IDVarchar2 (40)This field specifies the zone for which the count request was created. 
LOCATION_SIZE_CODEVarchar2 (40)This field defines the location capacity. This field maps to the primary key of the YFS_LOCATION_SIZE_CODE table. 
TRANSACTION_IDChar (40)This field specifies the transaction ID of the count request for which the count strategy is determined. 
TASK_TYPE_ITEM_CLASSIFICATION1Varchar2 (100)This field indicates the first item classification for determining the count strategy. 
TASK_TYPE_ITEM_CLASSIFICATION2Varchar2 (100)This field indicates the second item classification for determining the count strategy. 
TASK_TYPE_ITEM_CLASSIFICATION3Varchar2 (100)This field indicates the third item classification for determining the count strategy. 
REQUEST_TYPEVarchar2 (40)This field defines the count strategy differently for different request types. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.

MODIFYTSDateTimeThe timestamp for the last modification of this record. 
CREATEUSERIDVarchar2 (40)The user who created this record. 
MODIFYUSERIDVarchar2 (40)The user who last modified this record.

MODIFYPROGIDVarchar2 (40)The program that modified this record.
