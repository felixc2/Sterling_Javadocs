Entity Definition
YFS_VALIDATION
( CONFIGURATION Table )
This table stores the configuration for a validation



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription

A logical foreign key to the following table:

DESCRIPTIONVarchar2 (100)The description of the validation. 
EXTERNAL_INIT_PERFORMChar (1)The flag indicates that the initial validation is done outside of the framework.. 
VALIDATION_TYPEVarchar2 (10)The type of the validation. 
INPUT_TEMPLATEVarchar2 (40)The template name identfier is for determining template to be used in business API for sending the data in XML format to approval engine. 
MODEL_TEMPLATEVarchar2 (40)The model template is used in SBC for showing appropriate model in Approval Rule creation screens 
APPROVAL_POSTPONABLEChar (1)The flag indicates if the transaction violation can be approved at a later point of time. If it is marked as (N), then violation has to be approved before operator can proceed with the transaction. 
VIOLATION_ENTITYVarchar2 (100) This specifies the violation entity for storing the violations and approver information. 
ERROR_CODEChar (24)This specifies the error code that needs to thrown in case validation fails. 
PRE_DEFINED_GREEX_RULECLOBThe pre-defined greex rules for the validation. 
DATA_CAPTURECLOBThis is used to indicate the attributes whose values need to be captured to be used as part of the greex condition defined in the pre-defined greex rule. 
IS_SYSTEM_DEFINEDChar (1)This flag indicates that the validation is provided as part of the factory setup and should not be modified. 
IS_OVERRIDABLEChar (1)This flag indicates that the validation can be overridden by overridden rules. 
CONFIGURATION_GROUPVarchar2 (40)The configuration group that identify the level for which the validation can be configured. 
REAPPROVAL_REQUIREDChar (1)This flag specifies if each transaction violation for this validation needs to be approved or not. 
IS_USER_EXIT_ACTIVEChar (1)This flag indicates that user exit for the validation is active or not.
