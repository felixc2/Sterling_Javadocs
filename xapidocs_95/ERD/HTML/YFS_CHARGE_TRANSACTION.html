Entity Definition
YFS_CHARGE_TRANSACTION
( TRANSACTION Table )
This table stores information about financial transactions associated with order processing. Every action carried out on an order which may have an impact on financial components is logged in this table.

Records purged from this table are archived into YFS_CHARGE_TRANSACTION_H

This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

By default the identifier is same as primary key i.e Charge Transaction Key will be set as Async Request Identifier. This identifier can be used for recording the response to the asynchronous request.

This field should refer to the Async Request Identifier field of the original Charge Request that was marked as asynchronous. This field enables tracking recorded responses for asynchronous charge requests.
