Entity Definition
YFS_PROMOTION
( TRANSACTION Table )
This table stores information regarding promotions that are associated to an order or any of its lines.

Records purged from this table are archived into YFS_PROMOTION_H

You can extend this table by adding columns but not by creating hang-off table(s). Please refer to the Customization Guide for detailed procedures and restrictions.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
PROMOTION_KEYChar (24)The primary key for the YFS_PROMOTION table.  
PROMOTION_IDVarchar2 (100)This identifies the promotion that is added to this order. This will be used as the CouponID when determining the price of the order. 
ORDER_HEADER_KEYChar (24)The order on which this promotion is added.
A logical foreign key to the following tables:
  • YFS_ADDNL_LINE_PRICE (in combination with the ADDITIONAL_LINE_PRICE_KEY and ORDER_LINE_KEY columns)
  • YFS_ORDER_LINE (in combination with the ORDER_LINE_KEY and ADDITIONAL_LINE_PRICE_KEY columns)
  • YFS_ORDER_HEADER (in combination with the ADDITIONAL_LINE_PRICE_KEY and ORDER_LINE_KEY columns) 
ORDER_LINE_KEYChar (24)The order line on which this promotion is added.
A logical foreign key to the following tables:
  • YFS_ADDNL_LINE_PRICE (in combination with the ADDITIONAL_LINE_PRICE_KEY and ORDER_HEADER_KEY columns)
  • YFS_ORDER_LINE (in combination with the ORDER_HEADER_KEY and ADDITIONAL_LINE_PRICE_KEY columns)
  • YFS_ORDER_HEADER (in combination with the ORDER_HEADER_KEY and ADDITIONAL_LINE_PRICE_KEY columns) 
PROMOTION_TYPEVarchar2 (100)The type of promotion. Like DISCOUNT, BOGO. 
PROMOTION_GROUPVarchar2 (6)The group for which this promototion belongs to. Valid values: COUPON, MANUAL. The valid values are:
  • COUPON: Indicates that this promotion is intended for coupons.
  • MANUAL: Indicates that this promotion is intended for manual pricing rules. 
PROMOTION_APPLIEDChar (1)A flag indicating if the promotion is applied. When ValidateReferences=Y the value can be 'P' meaning the promotion was partially applied. 
DENIAL_REASONVarchar2 (200)If the promotion is denied, the reason for the denial. 
DESCRIPTIONVarchar2 (200)Description of the promotion. 
OVERRIDE_ADJUSTMENT_VALUENumber (19,6)For promotions, the overridden Adjustment value used in place of the configured Adjustment of the corresponding Pricing Rule Action. 
OVERRIDE_MAX_ADJUSTMENTNumber (19,6)For promotions, the overridden Maximum adjustment value for the pricing rule. 
VALIDATE_REFERENCES_VALUEChar (1)For Coupon promotions, determines whether the references will be validated or not 
IS_EXTERNALChar (1)This flag specifies whether the promotion is external. 
PROMOTION_CATEGORYVarchar2 (24)Specifies category of external promotion. Valid values are ORDER,ITEM and SHIPPING 
EXCLUSIVITY_TYPEVarchar2 (100)Exclusivity Type of the Promotion. Valid Values are: ALL, RULE_TYPE_ITEM, RULE_TYPE_ORDER, ONE_RULE_WITHIN_ID and GLOBAL_RULE 
EXCLUSIVITY_IDVarchar2 (100)ExclusivityID is used to group set of Promotions for mutual exclusivity. Exclusivity Type settings are applicable among Pricing Rules having same ExclusvityID. 
PRIORITYNumber (10,0)This attribute is used to prioritize Promotion among a list of applicable Promotions with same Exclusivity settings. Higher the value of this attribute, higher the priority of Promotion, to be applied to an order/item. 
ADDNL_LINE_PRICE_KEYChar (24)Primary key attribute of the YFS_ADDNL_LINE_PRICE table. 
CREATETSDateTimeTimestamp when the record was created. 
MODIFYTSDateTimeTimestamp when the record was last modified. 
CREATEUSERIDVarchar2 (40)User who created the record. 
MODIFYUSERIDVarchar2 (40)User who modified the record. 
CREATEPROGIDVarchar2 (40)Program that created the record. 
MODIFYPROGIDVarchar2 (40)Program that modified the record. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.
