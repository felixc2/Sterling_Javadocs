Entity Definition
YFS_BASE_TRANSACTION
( CONFIGURATION Table )
This table stores base transactions defined in the system.



You can extend this table by adding columns but not by creating hang-off table(s). Please refer to the Customization Guide for detailed procedures and restrictions.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
BASE_TRANSACTION_KEYChar (24)The primary key for the YFS_BASE_TRANSACTION table.  
BASE_PROCESS_TYPE_KEYChar (24)The base process type that is associated with this base transaction.
A logical foreign key to the following table:
  • YFS_BASE_PROCESS_TYPE 
ABSTRACT_TRANSACTIONChar (1)This flag indicates whether this transaction is an abstract transaction. 
BASE_TRANIDChar (24)The unique identifier for a base transaction. 
OWNER_KEYChar (24)The organization code that owns this transaction. 
BASE_TRANNAMEVarchar2 (50)The base transaction's name. 
WORKS_OFF_TASK_QChar (1)This flag indicates whether a task queue record needs to be created for this transaction. Tasks are created if a document reaches the status picked by this transaction. 
TASK_Q_KEY_TYPEVarchar2 (40)The type of tasks that this transaction processes. This field is used by the workflow engine to determine whether tasks needs be created for this transaction upon completion by other transaction. 
LISTENER_TYPEChar (2)The type of the listener transaction. Each process type to process type combination consistutes a different listener type. The valid values are:
  • 01: Indicates the chained order to sales order listener type.
  • 02: Indicates the derived order to sales order listener type.
  • 03: Indicates the shipment to order listener type.
  • 04: Indicates the load to shipment listener type.
  • 05: Indicates the receipt to shipment listener type.
  • 06: Indicates the receipt to order listener type.
  • 07: Indicates the work order to container listener type.
  • 08: Indicates the work order to order listener type. 
BASE_PROCESS_TYPE_LISTENED_TOChar (24)The base process type that a base transaction listens to for status changes. 
AGENT_JAVA_CLASSVarchar2 (128)The java class used for consuming work flow messages being sent to a pipeline. 
REQUIRES_DERIVED_DOC_TYPEChar (1)This field indicates whether derived document types are required. The valid values are:
  • N: Indicates that it is not required.
  • S: Indicates a single instance of each DERIVED_DOCUMENT_TYPE for a process type and base transaction.
  • M: Indicates multiple instances of each DERIVED_DOCUMENT_TYPE for a process type and base transaction. 
REQUIRES_CHAINED_DOC_TYPEChar (1)This field indicates whether chained document types are required. 
PICK_STATUS_FILTERChar (5)This field indicates how pickup statuses will be filtered. The valid values are:
  • ALL: Indicates that derived transactions can have any pickup statuses.
  • EXACT: Indicates that derived transactions can only have base pickup statuses.
  • BASE: Indicates that derived transactions can only have base pickup statuses. 
DROP_STATUS_FILTERChar (5)This field indicates how drop statuses will be filtered. The valid values are:
  • ALL: Indicates that derived transactions can have any drop statuses.
  • EXACT: Indiates that derived transactions can only have base drop statuses.
  • EXTN: Indicates that derived transactions can have base drop statuses or any of its extensions. 
SINGLE_DROP_STATUSChar (1)This flag indicates whether this base transaction can only have one drop status. 
EXTERNALLY_TRIGGERABLEChar (1)This flag indicates whether this base transaction is externally triggered. 
TIME_TRIGGERABLEChar (1)This flag indicates whether this base transaction is time-triggered. 
USER_TRIGGERABLEChar (1)This flag indicates whether this base transaction is user-triggered. 
HOLD_TYPE_ENABLEDChar (1)This flag indicates whether this transaction is capable of understanding hold types. Default value is 'N'. 
CAN_RESOLVE_HOLD_TYPEChar (1)This flag indicates whether this transaction is capable of resolving hold types. Default value is 'N'. 
SUPPORTED_DEPENDENCY_TYPEChar (1)This flag indicates what type of support this type of transaction has for transaction dependencies. 
SUPPORTED_COMPLETION_TYPEChar (1)This flag indicates if transaction completion can be recorded for this transaction. 
HOLD_PREVENTION_LEVELVarchar2 (40)The level at which a transaction can be prevented by a hold. Possible values are 'ORDER' and 'ORDER_LINE'. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications. 
CREATETSDateTimeTimestamp when the record was created. 
MODIFYTSDateTimeTimestamp when the record was last modified. 
CREATEUSERIDVarchar2 (40)User who created the record. 
MODIFYUSERIDVarchar2 (40)User who modified the record. 
CREATEPROGIDVarchar2 (40)Program that created the record. 
MODIFYPROGIDVarchar2 (40)Program that modified the record. 
SUPPORTS_OPERATION_LEVELChar (1)This is a boolean that identifies whether a transaction supports operation level.
