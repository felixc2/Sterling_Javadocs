Entity Definition
YFS_ACTIVITY_DEMAND
( TRANSACTION Table )
Demand on inventory may be placed for an activity from another activity group. This entity stores information about such demands placed.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns

NODE_KEYChar (24)The primary key for the YFS_ACTIVITY_DEMAND table. 
FROM_ACTIVITY_GROUPVarchar2 (40)This field identifies an activity group on which the demand is placed. The valid values are:

  • OUTBOUND_PICKING: Outbound Picking

  • OUTBOUND_INVENTORY_CONTAINER_BUILD: Outbound Inventory Container Build
  • SHIPPING_CONTAINER_BUILD: Shipping Container Build
  • SHIP: Shipping
  • INSPECTION: Inspection 
FOR_ACTIVITY_CODEVarchar2 (40)This field uniquely identifies an activity for which the demand is placed.

FOR_SHIPMENT_LINE_KEYChar (24)The shipment line key that is associated with this activity demand.
A logical foreign key to the following table:
  • YFS_SHIPMENT_LINE

A logical foreign key to the following table:

LPN_NOVarchar2 (40)The case or pallet License Plate Number. This key is a unique key for the YFS_LPN_HDR table. 
FOR_MOVE_REQUEST_LINE_KEYChar (24)The move request line key associated with this activity demand. 
ORDER_HEADER_KEYChar (24)Primary key of YFS_Order_Header table.
A logical foreign key to the following table:
  • YFS_ORDER_HEADER 
ORDER_NOVarchar2 (40)One of the unique identifiers of the order against which the demand is created. 
DOCUMENT_TYPEVarchar2 (40)The document type that is associated with the order against which the demand is created. For example, Sales Order document (0001), Planned Order document (0002), Return Order document (0003) , and Purchase Order document (0005). 
LPN_TYPEVarchar2 (40)This field indicates the LPN type to be putaway. At this time the supported values are Pallet and Case. The valid values are:
  • CASE: Case
  • PALLET: Pallet 
ORGANIZATION_CODEChar (24)The inventory organization against which the inventory is stocked for this item in the location inventory tables. Even though the Enterprise Code is passed as input in the activity demand APIs, it is not referenced to inventory consolidator. This is done because putaway is performed for the stock based on the demand and existing inventory. 
ITEM_IDChar (40)This field indicates the unique item identifier associated with the organization code. 
UOMVarchar2 (40)This field indicates the item's unit of measure. 
PRODUCT_CLASSChar (10)Inventory classification of an item based on product's characteristics. Typical values are FQ - First Quality, SQ - Second Quality etc. 
INVENTORY_STATUSChar (10)The unique key of the YFS_INVENTORY_STATUS table. This key refers to an inventory sub-classification based on the results of the inventory control processes within a warehouse. 
SEGMENT_TYPEVarchar2 (40)This field indicates that when inventory is customized, the inventory needs to be tracked separately for the buyer or buyer category for whom the inventory is customized. 
SEGMENTVarchar2 (40)This field indicates that when inventory is customized for a specific order, it needs to be tracked separately so that it can be allocated to that order. 
DEMAND_QTYNumber (14,4)This field specifies the demand quantity that should be placed in a location to perform the activity. 
SATISFIED_QTYNumber (14,4)This field specifies the actual quantity diverted till now. This quantity bucket gets updated every time a product is diverted for a specific purpose. 
DEMAND_LOCATION_IDVarchar2 (40)This field specifies the location identifier. This in conjunction with NODE_KEY identifies a unique location in the node, the location where an activity can be performed on the product. Typically, this field will be selected when the product should be placed in a specific location. For example, if the warehouse has just one QC location, then the quantity being requested should be placed in the QC location. 
DEMAND_SATISFIEDChar (1)This flag indicates whether the demand has been satisifed. This is automatically determined by the system based on demand quantities. 
IS_INFINITE_DEMANDChar (1)This flag indicates whether the demand is of a permanent nature. All permanent demands are never marked satisfied but purged when it is out of context. For e.g. when a shipment gets purged, all unsatisfied demands for the shipment are also purged. 
PRIORITYNumber (10,0)This field identifies the priority with which the demand is created. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.

CREATEUSERIDVarchar2 (40)The user who created this record. 
MODIFYUSERIDVarchar2 (40)The user who last modified this record.
