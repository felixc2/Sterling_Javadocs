Entity Definition
YFS_LOAD_STOP
( TRANSACTION Table )
Stores information about a load stop. A load stop is identified by a stop node - which is a receiving or shipping node on the system, or a stop address in cases where stop nodes do correspond to a receiving node which is not maintained on the system. A load stop is created and managed by the manageLoad API. Certain time related attributes such as actual arrival and departure dates, and the stop reached attribute are updated by the receiveIntransitUpdates API.

Records purged from this table are archived into YFS_LOAD_STOP_H

You can extend this table by adding columns but not by creating hang-off table(s). Please refer to the Customization Guide for detailed procedures and restrictions.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.
