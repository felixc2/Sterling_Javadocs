Entity Definition
YPM_PRICING_RULE_ACTION
( MASTER Table )
This table maintains the information about the actions of a pricing rule. One of the actions is fired depending on the condition satisfied. This table also maintains the tiered conditions of order total and item quantity rule.



You can extend this table by adding columns but not by creating hang-off table(s). Please refer to the Customization Guide for detailed procedures and restrictions.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
PRICING_RULE_ACTION_KEYChar (24)The primary key for the YPM_PRICING_RULE_ACTION table.  
PRICING_RULE_KEYChar (24)The rule this action is associated.
A logical foreign key to the following table:
  • YPM_PRICING_RULE 
ADJUSTMENT_TYPEVarchar2 (40)Type of adjustment. The valid values are:
  • FLAT_FEE: Applicable only for shipping order total rule. A flat fee action generates an adjustment such that the shipping charge is discounted to the value specified in the adjustment.
  • PERCENT: Indicates that the adjustment is a percentage on the unit price of an item or order total before shipping and tax.
  • ABSOLUTE: Indicates that the adjustment is a absolute value on the unit price of an item or order total before shipping and tax.
  • FIXED_LINE_PRICE: Indicates the rule is for fixed cost for a particular item or combination of multiple items.
  • ABSOLUTE_TOTAL: Applicable only for shipping item quantity rule. Indicates that the adjustment is a absolute value on the total shipping charge across all applicable items.
  • FLAT_FEE_TOTAL: Applicable only for shipping item quantity rule. A flat fee total action generates an adjustment such that the total shipping charge across all applicable items is discounted to the value specified in the adjustment. 
WHEN_TO_APPLYVarchar2 (40)Indicates how to apply the action. The valid values are:
  • SINGLE_ITEM: Indicates that the rule adjustment is applicable on one unit of target item.
  • ALL: Indicates that the rule adjustment is applicable on all units of target item.
  • EACH_COMBINATION: Indicates that the rule adjustment is applicable on the target item, as many times as the rule condition is satisfied. Applicable only for combination type of rules.
  • SPECIFIC_NUMBER: Indicates that the rule adjustment is applicable on specific units of target item 
ORDER_TOTALNumber (19,6)The Order total tier for the order total adjustments. Applicable only for rule type order total. 
QUANTITYNumber (14,4)Quantity of triggering item for tiered actions. Applicable only for rule type item quantity. 
ADJUSTMENTNumber (19,6)Adjustment to be applied as part of this action. 
MAX_MANUAL_ADJUSTMENTNumber (19,6)The maximum adjustment that can be applied as part of this manual rule action. 
CHARGE_CATEGORYVarchar2 (20)Unique identifier of the Charge Name. Category to which the charge name belongs to. 
CHARGE_NAMEVarchar2 (40)Name of the charge. 
MAX_ABSOLUTE_ADJUSTMENTNumber (19,6)Maximum absolute adjustment that can be provided for this pricing rule action within a transaction. 
TARGET_OCCURRENCE_LIMITNumber (14,4)Maximum Number of occurrences of adjustment because of this rule in an order. This field is populated WhenToApply is specified as SPECIFIC_NUMBER. 
TARGET_ITEM_DETER_RULEVarchar2 (20)Specifies on which target item the action should be applied. This is required while specifying apply on: 'one quantity of target item' or 'every combination' The valid values are:
  • LOW_PRICED_ITEM: Indicates that the adjustment is to be applied on item with the lowest unit price.
  • HIGH_PRICED_ITEM: Indicates that the adjustment is to be applied on item with the highest unit price.
  • SAME_LOW_PRICE_ITEM: Indicates that the adjustment is to be applied on item with the same or less unit price than any item on which the pricing rule is triggered.
  • ALL_ITEMS: This value is used when defining combination rules with target as 'All items in order', but excludes certain items. 
QUALIFIER_AMOUNT_TYPEVarchar2 (40)Specifies qualifier amount type of the action. Qualifier amount type can be based on minimum unit price or minimum total extended price. The valid values are:
  • TRIGGER_MINIMUM_UNIT_PRICE: This rule condition satisfies only when trigger item's unit price is greater or equal to the value in the QUALIFIER_AMOUNT column.
  • TRIGGER_MINIMUM_TOTAL_EXTENDED_PRICE: This rule condition satisfies only when the sum of trigger items' extended prices is greater or equal to the value in the QUALIFIER_AMOUNT column.
  • TRIGGER_MINIMUM_CUMULATIVE_EXT_PRICE: This rule condition satisfies only when the sum of the trigger items' extended prices plus any already applied discounts to the trigger items is greater or equal to the value in the QUALIFIER_AMOUNT column.
  • TARGET_MINIMUM_UNIT_PRICE: This rule condition satisfies only when the target item's unit price is greater or equal to the value in QUALIFIER_AMOUNT column.
  • TARGET_MINIMUM_TOTAL_EXTENDED_PRICE: This rule condition satisfies only when the sum of target items' extended prices is greater or equal to the value in the QUALIFIER_AMOUNT column.
  • TARGET_MINIMUM_CUMULATIVE_EXT_PRICE: This rule condition satisfies only when the sum of the target items' extended prices plus any already applied discounts to the target items is greater or equal to the value in the QUALIFIER_AMOUNT column. 
QUALIFIER_AMOUNTNumber (19,6)A number that needs to be satisfied in conjunction with the QUALIFIER_AMOUNT_TYPE for the rule condition to satisfy. 
APPLY_ADJ_TO_REMAINDERChar (1)Indicates that adjustments should be applied only to the remainder of the transaction. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.
