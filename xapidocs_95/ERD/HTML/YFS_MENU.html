Entity Definition
YFS_MENU
( CONFIGURATION Table )
Stores information on how a menu needs to be organized within an application.



You can extend this table by adding columns but not by creating hang-off table(s). Please refer to the Customization Guide for detailed procedures and restrictions.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
MENU_KEYChar (24)The primary key for the YFS_MENU table.  
PARENT_MENU_KEYChar (24)Parent menu item to which a menu item belongs.  
OWNER_KEYChar (24)This key indicates the owner of the menu. 
MENU_TYPEVarchar2 (10)Type of a menu. MENU/RESOURCE. RESOURCE typed menus are leaf nodes in the menu hierarchy that have an associated resource to access. Menu Items of type MENU are just place holders creating an organized menu tree.  
MENU_DESCRIPTIONVarchar2 (100)Description of a menu item. 
DISPLAY_MODEVarchar2 (40)Display mode in which a menu needs to be displayed. For example, Dialog, Docked, or HelpMenu.  
ICONVarchar2 (255)An Icon associated with a menu item. 
MENU_SEQNumber (5,0)A sequence number used for sorting child menu items below a particular menu item.  
RESOURCE_KEYChar (24)A resource associated with a menu item. This is populated only for leaf menu items.
A logical foreign key to the following table:
  • YFS_RESOURCE 
SYSTEM_DEFINEDChar (1)Indicates if a menu item was system defined.  
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.
