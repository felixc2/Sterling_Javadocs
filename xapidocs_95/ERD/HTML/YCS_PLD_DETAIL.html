Entity Definition
YCS_PLD_DETAIL
( CONFIGURATION Table )
This table stores information related to UPS PLDUpload, FedEx 020,120,007,107,023,123 transactions and Airborne shipment request, reply and void transactions. This information is as per the specification given by the respective carrier.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
PLD_DETAIL_KEYChar (24)The primary key for the YCS_PLD_DETAIL table.  
CARRIERVarchar2 (40)This is the adaptor key that is trying to find the mapping. 
TXNChar (30)The transaction codes like 020, 120, 007,107,023,123 for FedEx, PLDUpload for UPS, GENERATE_SHIPMENT, GENERATE_SHIPMENT_RESPONSE, VOID_SHIPMENT for AirBorne. 
SUB_TXNChar (30)The sub-transactions like Request, Reply for AirBorne and FedEx, PackageLevelDetail, AlternatePartyRecord, InternationalRecord, CommodityRecord, etc for UPS. 
FIELD_NAMEVarchar2 (200)The field name in the input XML like ShipperAccountNumber, ConsigneeCompanyName, etc. 
FIELD_LOCATIONNumber (9,0)Starting byte position of the field. 
FIELD_TYPEVarchar2 (15)The data type of the field. 
FIELD_LENGTHNumber (9,0)Length of the field. 
DECIMAL_DIGITSNumber (9,0)Indicates the number of bytes to the right of the decimal point in case of YCS_REAL datatype. 
REQUIRED_FLAGChar (1)Indicates whether the field is Required (R) or Optional (O) or Conditional (C). 
DEFAULT_TYPEVarchar2 (10)This field can be set to blanks or 'SYSTEM' or 'MAPPING' or 'USER'. If set to blanks, value for the field is taken from the corresponding field in the input XML. If this is set to 'SYSTEM', value for the field is taken from DEFAULT_VALUE. If this is set to 'MAPPING', value is taken from YCS_GENERAL_CODES table. If this is set to 'USER', value is taken from YCS_PLD_USER_DEFAULTS table. 
DEFAULT_VALUEVarchar2 (40)This field indicates the default value to be considered when DEFAULT_TYPE is set to 'SYSTEM'. 
CARRIER_FIELD_NAMEVarchar2 (200)This indicates the location of the field within the XML document. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.
