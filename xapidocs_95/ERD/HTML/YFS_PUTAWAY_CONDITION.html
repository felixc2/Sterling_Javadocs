Entity Definition
YFS_PUTAWAY_CONDITION
( CONFIGURATION Table )
Stores attributes values based on whose values put away will be carried. Allows setup at Item classification level, by inventory status (quality status), UOM etc. for a diversion type at the node enterprise level. Put away algorithm uses the best match algorithm to pick up the condition that matches the attributes of the stock that needs to be put away.

This table cannot be extended.

Sterling tables should be accessed only through Sterling APIs and Sterling Analytics. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of Sterling.
