Entity Definition
YFS_FLOW
( CONFIGURATION Table )
This table stores all the configured services. It is populated when configuring services using the service builder framework or when configuring time triggered transactions for agents.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
FLOW_KEYChar (24)The primary key for the YFS_FLOW table.  
GRAPH_UI_KEYChar (24)This identifier is a reference to the YFS_GRAPH_UI table, where the pictorial representation of the service is stored. 
TRANSPORT_TYPE_KEYChar (24)The transport type key of the beginning transport node for asynchronous services or LOCAL for synchronous services. 
OWNER_KEYChar (24)This key indicates the owner of the service. This value is "DEFAULT", which is the hub organization for all services. However, when moving event handlers and user exits, the user interface populates this with the appropriate enterprise organization key. 
FLOW_NAMEVarchar2 (128)The flow name is a unique identifier of the service. The field size is 128 bits to accommodate the backward compatibility naming format. 
FLOW_GROUP_NAMEVarchar2 (40)The service group name, which is used as a logical placeholder for the services. 
FLOW_TYPEChar (2)The flow type identifies the service associated with this table. The valid values are:
  • 00: Indicates that this service belongs to an Integration server. This is the default value.
  • 01: Indicates that this service belongs to an AgentServer. 
IS_REAL_TIMEChar (1)This flag determines whether a real time response is needed. The valid values are:
  • Y: Indicates a real time response is needed. In this case, an asynchronous transport node is not allowed in the service.
  • N: Indicates a real time response is not needed. 
IS_OUT_GOINGChar (1)This flag indicates whether the service is invoked synchronously or asynchronously. 
SYSTEM_DEFINEDChar (1)This flag indicates if the service is system defined and is modifiable. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications. 
CREATETSDateTimeTimestamp when the record was created. 
MODIFYTSDateTimeTimestamp when the record was last modified. 
CREATEUSERIDVarchar2 (40)User who created the record. 
MODIFYUSERIDVarchar2 (40)User who modified the record. 
CREATEPROGIDVarchar2 (40)Program that created the record. 
MODIFYPROGIDVarchar2 (40)Program that modified the record. 
PROCESS_TYPE_KEYChar (24)This key determines all the services belonging to some process type 
IS_PRINT_SERVICEChar (1)This field indicates if the service is used for printing the label using print document set APIs. 
DATA_LOADING_SERVICEChar (1)This Flag indicates that the service is for data loading
