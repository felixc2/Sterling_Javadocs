Entity Definition
YFS_EVENT_AUDITOR
( CONFIGURATION Table )
This table stores the configuration for order audit.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
EVENT_AUDITOR_KEYChar (24)The primary key for the YFS_EVENT_AUDITOR table.  
ORGANIZATION_CODEChar (24)The organization code assocaited with this event auditor.
A logical foreign key to the following table:
  • YFS_ORGANIZATION 
EVENT_AUDITOR_TYPEChar (24)Determines the type of auditor event this record corresponds to. Valid values are: TRANSACTION_EVENT, HOLD_ADDED_EVENT, HOLD_RESOLVED_EVENT, HOLD_REJECTED_EVENT, or APPLICATION_EVENT. 
TRANIDChar (40)If this auditor record corresponds to a transaction event, this column stores the transaction ID. 
EVENTIDVarchar2 (40)If this auditor record corresponds to a transaction event, this column stores the event ID.. 
HOLD_TYPEChar (20)If this auditor record corresponds to a hold event, this column stores the hold type that this record corresponds to. 
WIZARD_IDVarchar2 (255)If this auditor record corresponds to a wizard id, when a wizard closes a note with the provided reason and note text will automatically be logged against the Order Object 
APPLICATION_CODEVarchar2 (50)This is the corresponding application code for the wizard id provided and is used to add notes on confirmation of a wizard flow. 
NOTE_TYPEVarchar2 (40)The note type that will be logged in the notes table (stored in the notes table as ReasonCode). 
NOTE_TEXTVarchar2 (2000)If provided, this field will be used to set the text in a note. 
TRANSFORMATION_FILEVarchar2 (100)The XSL file that will be used to transform the XML data to the notes text. 
ACTIVEChar (1)Y/N flag indicating if this auditor event is active and should perform the action it corresponds to.

MODIFYUSERIDVarchar2 (40)User who modified the record. 
CREATEPROGIDVarchar2 (40)Program that created the record. 
MODIFYPROGIDVarchar2 (40)Program that modified the record. 
LOCKIDNumber (5,0)An integer used to track and prevent concurrent modifications.
