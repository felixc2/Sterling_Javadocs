Entity Definition
YFS_WAVE
( TRANSACTION Table )
This table stores wave definitions.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription

WAVE_NOVarchar2 (40)The wave number is a one up number that system generates to uniquely identify the wave in a node. 
NODE_KEYChar (24)The primary key for the YFS_WAVE table.

DESCRIPTIONVarchar2 (200)A brief description of the wave.

A logical foreign key to the following table:

DOCUMENT_TYPEVarchar2 (40)This field indicates the document type to which the wave belongs. 
PIPELINE_KEYChar (24)The pipeline key that is associated with the wave. 
WAVE_STATUSChar (40)This field indicates the status the wave is currently in. 
BG_WAVE_STATUSChar (40)This field indicates the status of the wave, when processed in background. Possible values are blank (not running in background), 00 (Cancellation Requested) and 01 (Cancellation Failed). 
RELEASE_NO_EARLIER_THANDateTimeThis field indicates that the wave is not released for picking before this date. 
RELEASE_NO_LATER_THANDateTimeThis field indicates that the wave is not released for picking after this date. 
SEQ_NONumber (5,0)This field indicates the sequence number associated with the wave. 
PRIORITYNumber (10,0)This field indicates the priority to apply to the wave. All waves will be processed in the order of this priority. 
REASON_CODEVarchar2 (40)This field indicates the reason code for modifying the wave. 
REASON_TEXTVarchar2 (255)This field describes any additional information as to why the wave was modified. 
OVERRIDE_RELEASE_DATEChar (1)This flag indicates whether the wave overrides the wave release date. If 'Y' is selected, the Assign Pick Location (Release Wave) agent ignores the Release No Earlier Than and Release No Later Than date ranges. 
OVERRIDE_CONCURRENT_WAVEChar (1)This flag indicates whether the wave overrides the concurrent wave that is released. If 'Y' is selected, the Assign Pick Location (Release Wave) agent ignores the number of waves released concurrently that is defined for a specific shipment group. 
ESTIMATED_MINUTESNumber (5,0)Indicates the number of minutes it is estimated that the wave would take to execute.
