Entity Definition
YFS_INVENTORY_DEMAND_TYPE
( CONFIGURATION Table )
This table stores valid demand types for the system.



This table is not extensible.

Tables should be accessed only through exposed APIs and Business Intelligence. Accessing them directly may result in issues during application of hot-fixes, service packs, and during upgrade to future releases of applications.

Columns
Column NameData TypeDescription
INVENTORY_DEMAND_TYPE_KEYChar (24)The primary key for the YFS_INVENTORY_DEMAND_TYPE table.  
DEMAND_TYPEVarchar2 (40)A Demand type code. 
DESCRIPTIONVarchar2 (100)Description of a demand type. 
PROMISED_DEMANDChar (1)Indicates if a demand type is used for promised demands. 
ALLOCATED_DEMANDChar (1)Indicates if this demand type is used for allocated demands. 
COMMUNICATED_DEMANDChar (1)This flag indicates if this demand type is used for demands, which have been communicated to external systems. 
PRIORITYNumber (10,2)Priority associatied with a demand type. Valid values are 1, 2, and 3. Value 1 represents the highest priority demand type and value 3 represents the lowest priority demand type. Demands will be allocated to the supplies in the order of their demand type priorities. 
SYSTEM_DEFINEDChar (1)Set to 'Y' if this supply type is a factory setting at product release time.
